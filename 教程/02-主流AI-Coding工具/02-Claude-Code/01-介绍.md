# 2.2.1 Claude Code 介绍

> Claude Code 不是又一个「AI 聊天工具」——它是能自主读文件、写代码、跑命令、管理 Git 的终端智能体。被 46% 的开发者选为最爱工具，代码质量评分 9.7/10。

---

## 一句话理解

Claude Code 是 Anthropic 推出的**终端 CLI AI 编程助手**。只需描述需求，它就能自动计划并执行：读文件、写代码、跑命令、管理 Git、提交 PR。

---

## Claude Code vs 普通 AI Chat

这是理解 Claude Code 最关键的一张对比表：

| 维度 | 普通 AI Chat（豆包、DeepSeek 网页版） | Claude Code |
|---|---|---|
| 交互方式 | 一问一答 | 规划 + 自主执行 |
| 文件操作 | 不支持 | 读写项目文件 |
| 命令执行 | 不支持 | 运行脚本、测试、部署 |
| Git 管理 | 不支持 | commit、PR、分支操作 |
| 扩展能力 | 有限 | MCP + Skills 生态 |
| 上下文 | 几十 K | 200 万 Token |

> 💡 **关键差异**：普通 AI Chat 只能「告诉你答案」，Claude Code 能「帮你把事情做了」。

---

## 四层能力体系

Claude Code 的设计哲学是「分层协作」。理解这四层，你就理解了它的全部：

```
┌────────────────────────────────┐
│  Prompt（指令层）               │ ← 自然语言描述需求
├────────────────────────────────┤
│  Skill（方法论层）              │ ← 可复用的专业能力包
├────────────────────────────────┤
│  Project（上下文层）            │ ← 持久化工作空间
├────────────────────────────────┤
│  MCP（工具连接层）              │ ← 连接外部数据源和工具
└────────────────────────────────┘
```

### 每一层的定位

- **Prompt**：基础指令。门槛最低，但复用性差。每次都要重新描述。
- **Skill**：可复用的方法论。把专业知识编码为 AI 能遵循的流程文档。
- **Project**：持久化上下文。通过 `CLAUDE.md` 让 AI 记住项目规范，跨会话不丢失。
- **MCP**：外部工具连接。让 AI 能操作浏览器、数据库、GitHub、Figma 等外部系统。

---

## 三种运行模式

Claude Code 提供三种模式，适配不同场景：

| 模式 | 如何进入 | 行为 | 适合场景 |
|---|---|---|---|
| **默认模式** | 正常启动 | 修改前询问确认 | 日常开发（安全第一） |
| **自动模式** | 输入 `accept edits on` | 直接执行修改 | 信任度高的重复操作 |
| **规划模式** | 输入 `plan mode on` | 只讨论不修改 | 架构设计、方案讨论 |

### 实操建议

```
启动 → 默认模式（先安全）
  ↓ 熟悉后
大改动前 → 先切规划模式讨论方案
  ↓ 确认方案后
执行阶段 → 自动模式（减少确认打断）
  ↓ 关键操作前
回到默认模式 → 确保安全
```

## 适用场景

Claude Code 不只是写代码工具，它是一个通用的项目助手：

| 场景 | 典型任务 |
|---|---|
| 💻 软件开发 | 全栈项目、Bug 修复、重构、代码审查 |
| 🎨 前端设计 | 设计稿转代码、UI 生成、设计系统管理 |
| 📊 数据分析 | 文件处理、数据可视化、报告生成 |
| 🔧 DevOps | CI/CD 配置、部署脚本、监控设置 |
| 📝 文档写作 | 技术文档、API 文档、知识库管理 |

## Claude Code 的定位是什么

在整个 AI Coding 工具版图中，Claude Code 的定位是：

```
IDE 工具（Cursor/Trae）：日常编码，实时代码补全
    +
Claude Code：深度推理，架构攻坚，复杂任务
    +
云端 Agent（Codex）：自动化脚本，无人值守
```

**它不是 IDE 替代品，而是 IDE 的强力补充。** 不要在 Claude Code 里写每一行代码——那是 Cursor/Trae 的活。把 Claude Code 留给最需要深度思考的任务。

## 本章小结

1. Claude Code 是**终端 CLI 智能体**，不是聊天工具
2. 核心能力：自主读文件 → 写代码 → 跑命令 → 管 Git
3. 四层体系：Prompt → Skill → Project → MCP
4. 三种模式：默认（安全）、自动（高效）、规划（设计）
5. 定位：不是 IDE 替代品，是强力补充——留给最需要深度推理的任务
