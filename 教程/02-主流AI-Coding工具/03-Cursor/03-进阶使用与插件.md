# 2.3.3 Cursor 进阶使用与插件

> Background Agents、多模型策略、.cursorrules 优化、推荐插件——这些是让 Cursor 从「好用」到「生产力飞跃」的关键。

---

## Background Agents（Cursor 3）

Cursor 3 最大的新功能：Agent 可以在云端后台运行。

### 传统 Agent vs Background Agent

```
传统 Agent：
  发起任务 → IDE 被占用 → 等待完成 → 才能继续工作

Background Agent：
  发起任务 → Agent 在云端运行 → IDE 不受影响
  → 完成后通知你 → Review 结果
```

### 使用场景

| 场景 | 说明 |
|---|---|
| 代码重构 | 「把 src/components/ 下所有 class 组件改成函数组件」→ Agent 后台跑，你继续写新功能 |
| 测试生成 | 「给 src/api/ 下所有接口生成单元测试」→ 后台生成，完成后 Review |
| 代码审查 | 「检查 src/ 下所有文件是否有未处理 Promise」→ 后台扫描 |

### 最佳实践

```
1. 起步前写好清晰的 Prompt（越具体越好）
2. 不要同时启动多个 Agent（会冲突）
3. 完成后仔细 Review 每个改动
4. 如果有问题，直接在结果上 Cmd+K 修正
```

## 多模型策略

### 模型选择

Cursor 支持多种模型，合理切换可以优化成本和效果：

| 任务类型 | 推荐模型 | 原因 |
|---|---|---|
| 日常补全 | 默认（Sonnet / GPT-4o-mini） | 快、便宜、够用 |
| 代码生成 | Claude Sonnet | 代码质量最好的日常模型 |
| 复杂逻辑 | Claude Opus / GPT-5 | 深度推理能力 |
| 快速修改 | Haiku / GPT-4o-mini | 成本低、响应快 |
| 代码审查 | Claude Opus | 能发现深层问题 |

### 配置自定义 API

```
Settings → Models → Add Custom Model
  → 填入 API endpoint 和 key
  → 可使用国产模型（DeepSeek、GLM 等）
```

## .cursorrules 最佳实践

### 规则文件的结构

```markdown
# .cursorrules

## 技术栈
- 框架：Next.js 14 (App Router)
- 语言：TypeScript 严格模式
- 样式：Tailwind CSS + shadcn/ui
- 数据库：Prisma + PostgreSQL

## 编码规范
- 组件：Server Components 优先
- 函数：必须有完整类型注解
- 文件命名：kebab-case
- 导入：使用绝对路径 @/ 别名

## 禁止事项
- 不使用 any 类型
- 不直接操作 DOM
- 不使用 CSS-in-JS

## 项目结构
- src/app/ — 页面路由
- src/components/ — 共享组件
- src/lib/ — 工具函数
- src/types/ — 类型定义
```

### 写作原则

1. **控制在 50 行以内**——太长的规则 AI 也记不住
2. **写约束，不写教程**——AI 需要边界，不需要被教育
3. **定期更新**——项目演变时同步更新规则
4. **只写项目特有规范**——通用知识（如 TypeScript 语法）不需要写

## 推荐插件

Cursor 完全兼容 VS Code 插件生态。以下是特别推荐的高效插件：

| 插件 | 用途 | 推荐理由 |
|---|---|---|
| **GitLens** | Git 增强 | 行级 blame、历史对比、分支可视化 |
| **Prettier** | 代码格式化 | 配合 .prettierrc 自动统一格式 |
| **Tailwind CSS IntelliSense** | Tailwind 提示 | 自动补全 class、颜色预览 |
| **Error Lens** | 行内错误显示 | 错误信息直接显示在代码行末 |
| **Thunder Client** | API 测试 | 轻量级 Postman 替代 |

### 安装插件

```
1. Cmd/Ctrl + Shift + X 打开扩展面板
2. 搜索插件名
3. 点击 Install
```

## 工作流建议

### 日常开发节奏

```
1. Tab 补全：写代码时自然触发
2. Cmd+K：需要对代码块做精确修改时
3. Chat：不确定怎么做，先讨论
4. Composer：大任务、跨文件修改
5. Background Agent：耗时任务放后台
```

### Cursor 定位

Cursor 是「日常 IDE」——适合作为主力编辑器。它的定位不是替代 Claude Code，而是替代 VS Code。

```
Cursor：日常编码（占 80%）
Claude Code：架构攻坚（占 15%）
Codex：自动化脚本（占 4%）
```

## 本章小结

1. Background Agents 让你在 Agent 工作时不受打扰
2. 不同模型用于不同任务——简单任务用便宜模型，复杂任务用贵模型
3. .cursorrules 控制在 50 行以内，写约束不写教程
4. Cursor = VS Code 生态 + AI Agent 能力
5. Cursor 是日常 IDE，Claude Code 是攻坚工具
