# 2.4.3 Trae 进阶使用与对比

> MCP 配置、火山引擎 Coding Plan、Trae vs Cursor 最终选择指南——这些是让 Trae 从「够用」到「好用」的关键。

---

## MCP Server 配置

Trae 支持 MCP（Model Context Protocol），可接入自定义工具扩展 AI 能力。

### 配置文件

在项目根目录创建 `.trae/mcp.json`：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "github": {
      "command": "npx",
      "args": ["@anthropic/mcp-github"]
    }
  }
}
```

### 支持的 MCP 类型

与 Claude Code 的 MCP 生态兼容。常用的包括：

| MCP | 用途 |
|---|---|
| Playwright | 浏览器自动化测试 |
| GitHub | PR/Issue 管理 |
| Filesystem | 文件系统操作 |
| Context7 | 最新框架文档 |

> 💡 Trae 的 MCP 支持在快速迭代中，部分复杂 MCP 可能兼容性不如 Claude Code。

## 火山引擎 Coding Plan

如果需要使用商业级模型（DeepSeek-V3.2、Kimi-K2.6、MiniMax 等）：

### 接入步骤

1. **注册火山引擎账号**：访问火山引擎官网
2. **开通 Coding Plan**：在控制台找到 Coding Plan 产品
3. **获取 API Key**：创建 API Key 并妥善保存
4. **配置到 Trae**：

```
设置 → 模型配置 → 自定义 API
  ├─ Base URL：火山引擎提供的端点
  ├─ API Key：你的密钥
  └─ 选择模型：deepseek-v3.2 / kimi-k2.6 / minimax-latest
```

### Coding Plan 的优势

| 维度 | Trae 内置模型 | Coding Plan |
|---|---|---|
| 模型选择 | 有限 | 20+ 商业模型 |
| 代码质量 | 8.8/10 | 接近 Claude Code |
| 成本 | 免费 | 按量付费 |
| 适合 | 日常开发 | 重要项目攻坚 |

## 项目规则配置

### .trae/rules.md

Trae 的项目规则文件，等价于 Claude Code 的 CLAUDE.md：

```markdown
# 项目规则

## 技术栈
- 前端：React 18 + TypeScript + Vite
- 样式：Tailwind CSS
- 状态管理：Zustand
- 路由：React Router v6

## 编码规范
- 组件使用函数式 + Hooks
- 文件命名：PascalCase（组件）/ camelCase（工具函数）
- 每个组件一个文件
- 导出使用 named export

## 禁止
- 不使用 class 组件
- 不直接操作 DOM（除非必要）
- 不使用 any 类型

## 目录结构
- src/components/ — UI 组件
- src/pages/ — 页面组件
- src/hooks/ — 自定义 Hooks
- src/utils/ — 工具函数
```

## Trae vs Cursor 终极对比

### 快速决策表

| 场景 | 推荐 | 理由 |
|---|---|---|
| 预算为零 | **Trae** | 永久免费 |
| 国内中文项目 | **Trae** | 中文 98%，国内极速 |
| 追求最强模型 | **Cursor** | Claude Opus/GPT-5 更强 |
| 国际化/英语项目 | **Cursor** | 国际化生态更成熟 |
| VS Code 生态 | 两者均可 | 都兼容 VS Code 插件 |
| 云端 Agent | **Cursor 3** | Background Agents |
| 数据合规 | **Trae** | 国内服务器 |
| 学习入门 | **Trae** | 免费无压力 |

### 详细维度对比

| 维度 | Trae | Cursor |
|---|---|---|
| 定价 | **免费** | $20-200/月 |
| 中文理解 | **98%** | 75% |
| 国内速度 | **极速** | 一般，部分模型需代理 |
| 基座模型 | 国产模型 | Claude/GPT（更强） |
| Agent 能力 | Builder + SOLO | Composer + Background Agents |
| 国际化生态 | 弱 | 强 |
| 社区资源 | 中文为主 | 英文为主 |
| 适合人群 | 国内开发者 | 全球开发者 |

## 选 Trae 的理由

1. **你是国内开发者**：中文 98%，极速直连，数据合规
2. **你预算为零**：永久免费，没有隐藏收费
3. **你在学习阶段**：免费无压力，中文文档友好
4. **你的项目以中文为主**：中文理解能力远超其他工具
5. **你不需要顶级模型**：国产模型对大多数需求来说已经足够

## 选 Cursor 的理由

1. **你需要最强模型**：Claude Opus 和 GPT-5 的代码质量更高
2. **你的项目是国际化的**：Cursor 生态和社区以英文为主
3. **你需要 Background Agents**：Cursor 3 的云端 Agent 是独有功能
4. **你愿意付费**：$20/月的 Pro 计划性价比不错

---

## 组合策略

你不需要二选一。最佳策略是组合使用：

```
日常开发（80%）：Trae
  ↓ 遇到复杂推理需求时
切换到 Cursor 或 Claude Code
  ↓ 完成攻坚后
回到 Trae
```

这种组合充分利用了 Trae 的免费优势和 Cursor/Claude Code 的深度能力。

## 本章小结

1. Trae 支持 MCP 配置，可扩展 AI 能力
2. 火山引擎 Coding Plan 让你接入 20+ 商业模型
3. `.trae/rules.md` 是项目规则文件，让 AI 理解你的规范
4. Trae vs Cursor 不是「谁更好」，而是「谁更适合你」
5. 最佳策略：Trae 日常 + Cursor/Claude Code 攻坚
