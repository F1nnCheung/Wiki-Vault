# 2.3.1 Cursor 介绍与安装

> Cursor 是 AI 原生 IDE 的标杆——基于 VS Code，估值 290 亿美元。如果你从 VS Code 迁移，Cursor 是体验最平滑的选择。

---

## 概述

Cursor 由 Anysphere 开发，2026 年估值 290 亿美元，被 19% 的开发者选为最爱工具。它的核心优势是「在熟悉的 IDE 体验中加入 Agent 能力」——既有实时的 Tab 补全，也有跨文件的 Composer Agent。

**官网**：cursor.com

---

## 核心特点

| 特点 | 说明 |
|---|---|
| 基于 VS Code | 完全兼容 VS Code 插件和设置 |
| Tab 补全 | <100ms 延迟，多行代码预测 |
| Composer | 跨文件 Agent 编辑 |
| Cmd+K | 选中代码，自然语言修改 |
| Background Agents | Cursor 3 云端后台任务 |
| 多模型 | 支持 Claude、GPT、Gemini 等 |

---

## 安装

### 系统要求

| 系统 | 要求 |
|---|---|
| Windows | Windows 10/11，64 位 |
| macOS | 12.0+（Intel / Apple Silicon） |
| Linux | AppImage / deb / rpm |

### 安装步骤

1. 访问 **cursor.com** 下载安装包
2. 安装后首次启动，可选择从 VS Code 导入配置
3. 注册/登录 Cursor 账号
4. 选择订阅计划：

| 计划 | 价格 | 说明 |
|---|---|---|
| **Hobby** | 免费 | 基础功能，有限的模型调用额度 |
| **Pro** | $20/月 | 更多模型调用，高级功能 |
| **Business** | $40/月 | 团队协作，管理后台 |

> 💡 **建议**：先用 Hobby 免费版体验核心功能，确认适合后再升级 Pro。

### 从 VS Code 迁移

Cursor 的设计目标之一就是让 VS Code 用户零成本迁移：

```
安装 Cursor → 首次启动 → "Import from VS Code"
  → 自动导入：设置、快捷键、插件、主题
```

迁移后几乎不需要任何适应期。

---

## 订阅选择建议

### Hobby（免费版）够用吗？

适合：
- 想先体验 AI IDE 的用户
- 偶尔使用 AI 功能的轻度用户
- 预算有限的个人开发者

限制：
- 模型调用次数有限
- 高级模型（Opus、GPT-5）不可用

### Pro（$20/月）值得吗？

适合：
- 日常重度使用 AI 编程的开发者
- 需要用 Composer 做跨文件编辑
- 需要 Background Agents 后台任务

> 如果你的 AI 编程日均使用超过 2 小时，Pro 非常值得。

## 初次配置

安装完成后，建议完成以下初始化：

### 1. 配置模型

```
Settings → Models
  ├─ 启用 Claude Sonnet（默认，推荐）
  ├─ 可选 Claude Opus（更强，Pro 专属）
  ├─ 可选 GPT-4o（替代选择）
  └─ 可选 Gemini 2.5（谷歌生态）
```

### 2. 创建 .cursorrules

在项目根目录创建规则文件，让 AI 理解你的项目规范：

```markdown
# .cursorrules
你是一个 TypeScript 全栈开发者。
技术栈：Next.js 14 + Prisma + Tailwind CSS
编码规范：
- 所有函数必须有类型注解
- 组件使用 Server Components 优先
- API 路由使用 Route Handlers
```

### 3. 配置全局 Rules for AI

```
Settings → Rules for AI
```
全局规则对所有项目生效，适合放通用偏好（代码风格、语言偏好等）。
