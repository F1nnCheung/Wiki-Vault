# 2.3.2 Cursor 基本使用

> 四个核心功能 + 四个快捷键。掌握这些，你就能在日常开发中高效使用 Cursor。关键是理解每个功能的最佳场景。

---

## 概述

Cursor 的核心交互围绕四个功能展开：Tab 补全、Cmd+K、Composer、Chat。每个功能解决不同粒度的问题——从「补全一行」到「改造整个项目」。

---

## 功能一：Tab 补全

### 是什么

输入代码时 AI 自动预测并建议补全。延迟 <100ms，几乎无感。

### 能做什么

```
不只是补全一行——能预测多行代码：
- 函数体自动补全
- 条件分支推测
- 错误处理代码生成
- 测试用例自动生成
```

### 怎么用

```
写代码 → 灰字出现（AI 建议）→ Tab 接受 → 继续写
                                  → Esc 拒绝 → 继续写
```

### 最佳场景

- ✅ 写样板代码（CRUD 操作、表单处理）
- ✅ 重复模式（类似结构多次出现时）
- ✅ 函数参数补全（减少查文档）
- ❌ 复杂算法逻辑（交给 Composer）
- ❌ 需要跨文件理解的任务（用 Chat 或 Composer）

---

## 功能二：Cmd+K — 内联编辑

### 是什么

选中一段代码，用自然语言描述想要的修改，AI 在 Diff 视图中展示结果。

### 怎么用

```
1. 选中要修改的代码
2. Cmd/Ctrl + K
3. 输入：「添加 try-catch 错误处理」
4. AI 生成修改 → Diff 预览
5. Accept（接受）或 Reject（拒绝）
```

### 使用技巧

```
✅ 好的 Cmd+K Prompt：
"把 fetch 改成 axios，加上超时 10s 和重试 3 次"

❌ 差的 Cmd+K Prompt：
"改一下"（太模糊）
```

### 最佳场景

- ✅ 单文件内的小范围修改
- ✅ 代码重构（提取函数、改命名）
- ✅ 添加错误处理、日志、注释
- ❌ 跨文件修改（用 Composer）
- ❌ 需要理解项目上下文的任务（用 Chat）

---

## 功能三：Composer — 跨文件 Agent

### 是什么

Cursor 的 Agent 模式。描述一个任务，AI 自动跨文件搜索、编辑、创建。

### 怎么用

```
1. Cmd/Ctrl + I 打开 Composer
2. 描述任务：
   "给所有 API 调用加上统一的错误处理和超时机制"
3. AI 自动搜索相关文件 → 跨文件修改
4. 逐个文件 Review → Accept
```

### Composer vs Cmd+K

| | Cmd+K | Composer |
|---|---|---|
| 作用范围 | 选中的代码块 | 整个项目 |
| 文件操作 | 仅当前文件 | 跨文件 |
| 启动方式 | 先选中，再命令 | 先命令，AI 自己找 |
| 适合 | 精确修改 | 大任务 |

### 使用技巧

```
✅ 好的 Composer Prompt：
"给 src/api/ 下所有请求函数加上超时配置和统一错误处理，
错误类型参考 types/errors.ts"

❌ 差的 Composer Prompt：
"改进错误处理"（太模糊，AI 不知道改什么）
```

### 最佳场景

- ✅ 跨文件的功能实现（如「给所有页面加上加载状态」）
- ✅ 代码迁移（如「把 class 组件改成 hooks」）
- ✅ 批量修改（如「给所有 API 调用加超时」）
- ❌ 精确小修改（用 Cmd+K 更高效）
- ❌ 探索性问题（用 Chat 先讨论）

---

## 功能四：Chat — AI 对话

### 是什么

侧边栏的 AI 对话面板，可以讨论代码、提问、获取建议。

### 怎么用

```
1. Cmd/Ctrl + L 打开 Chat
2. 输入问题或选中代码后提问
3. AI 回答，可 Apply 代码建议
```

### Chat vs Composer

```
Chat：讨论 → 理解 → 手动执行
Composer：下指令 → AI 自动执行

不确定怎么做时 → 先用 Chat 讨论
明确知道要做什么时 → 直接用 Composer
```

---

## @ 引用技巧

这是 Cursor 最强大的上下文控制功能：

| 引用 | 语法 | 效果 |
|---|---|---|
| 单文件 | `@file` | 引用一个文件的完整内容 |
| 文件夹 | `@folder` | 引用整个文件夹的结构 |
| 网页 | `@web` | 引用网页内容（文档、博客） |
| 官方文档 | `@docs` | 引用框架官方文档 |
| 整个代码库 | `@codebase` | 引用代码库用于全项目搜索 |

### 实战组合

```
在 Chat 中输入：
"@src/api/user.ts 这个文件有什么安全问题？
 参考 @docs Next.js 的安全最佳实践"

在 Composer 中输入：
"参照 @src/types/api.ts 的类型定义，
 重构 @folder src/api/ 下所有请求函数"
```

## 快捷键速查

| 功能 | macOS | Windows/Linux |
|---|---|---|
| Tab 补全 | `Tab` | `Tab` |
| 内联编辑 | `Cmd + K` | `Ctrl + K` |
| Composer | `Cmd + I` | `Ctrl + I` |
| Chat | `Cmd + L` | `Ctrl + L` |
| @ 引用 | `@` | `@` |

---

## 四种功能的决策矩阵

```
你需要什么？
├── 补全当前行 → Tab
├── 修改选中代码 → Cmd+K
├── 跨文件执行任务 → Composer
└── 讨论/提问/理解代码 → Chat
```

---

## 本章小结

1. **Tab**：最快的补全，适合写重复代码
2. **Cmd+K**：精确修改，先选中再命令
3. **Composer**：跨文件 Agent，描述任务让 AI 自动执行
4. **Chat**：先讨论再行动，适合不确定的场景
5. **@ 引用**：精确控制上下文，大幅提升 AI 理解准确度
