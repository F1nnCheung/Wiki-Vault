# 2.5.1 Codex 介绍与安装

> Codex 已从「云端编程助手」演化为统一的 AI Agent 操作平台——集编程、浏览器操作、电脑自动化、AI 生图于一体。四大产品形态（App/CLI/IDE/Cloud）共享同一套配置，配一次四处通用。

---

## 概述

2026 年的 Codex 是 OpenAI 推出的 **通用 AI Agent 操作平台**，与 Claude Code 并称 AI 编程双强。在 500+ 人开发者调查中 65% 日常更倾向使用 Codex——关键原因是 token 效率约为 Claude Code 的 4 倍，同样的活烧的额度少。

> **气质差异**：Claude Code 像一个边干边跟你商量的协作者；Codex 像一个「别打扰我，干完给你看」的执行型工程师。

---

## 四大产品形态

| 形态 | 适用人群 | 特点 |
|------|----------|------|
| **Codex App** | 零基础、桌面用户 | 图形界面，三栏布局，功能最全 ⭐推荐新手 |
| **Codex CLI** | 终端用户、开发者 | 命令行 Agent，支持交互和非交互（`codex exec`）模式 |
| **IDE Extension** | VS Code / Cursor / Windsurf 用户 | 嵌入编辑器，在熟悉环境中使用 |
| **Cloud (Web)** | 进阶用户、团队 | 任务丢 OpenAI 云端跑，干完自动开 PR，适合「过夜班」 |

> 四个形态 **共享同一套配置**（`~/.codex/config.toml`），配一次四处通用。

## 安装与配置

### App 安装（推荐新手）

1. 访问 [Codex 官网](https://chatgpt.com/codex/)，下载对应系统安装包
2. Windows 和 Mac 均支持（Mac 额外支持 Computer Use）
3. 启动 → ChatGPT 账户登录 → 选择工作类型 → **设置沙盒**（关键步骤）
4. 推荐预装：Git + Node.js + VSCode

**一键搬家**：Codex 支持从 Claude Code 和 Cowork 直接导入所有配置，降低迁移成本。

### CLI 安装（开发者推荐）

```bash
# npm 安装
npm i -g @openai/codex

# 或 Homebrew（Mac）
brew install --cask codex

# 启动并登录
codex

# 非交互模式（脚本/CI 中使用）
codex exec "把所有 console.log 删掉"
codex exec "任务描述" --json  # 输出 JSON 格式
```

### DeepSeek 模型接入

国内用户可通过 `mimo2codex + CC Switch` 方案接入 DeepSeek 模型：

```bash
# 安装中间件
npm install -g mimo2codex

# 启动本地转发服务
export DEEPSEEK_API_KEY="sk-your-key"
mimo2codex --model ds --port 8789

# 在 CC Switch 中配置供应商
# Base URL: http://127.0.0.1:8789/v1
# API Key: mimo2codex-local
```

详见 [[Wiki/wiki/topics/codex-deepseek-integration|Codex + DeepSeek 接入完整教程]]。

## 沙箱安全（首次启动必设）

Codex 的权限系统围绕**沙箱**展开，使用操作系统底层机制实现硬性限制：

**沙箱模式**（管它能动什么）：
- `read-only`：只能读 → 读代码、做规划
- `workspace-write`（默认）：读写项目目录 + 跑命令 → 日常开发
- `danger-full-access`：无边界 → 仅一次性容器

**审批策略**（管它什么时候问）：
- `untrusted`：啥都要问
- `on-request`：敏感操作才问 ⭐推荐
- `never`：永远不问

**推荐配置**：workspace-write + on-request + 自动审查
