# 3.3 OpenClaw vs Hermes 深度对比

> 两个 Agent 框架回答的根本问题不同：OpenClaw 关注「怎么安全可靠地执行」，Hermes 关注「怎么越变越强」。这个底层分歧导致了架构设计的完全分化。

---

## 核心差异一览

| 维度 | Hermes Agent | OpenClaw |
|---|---|---|
| **核心问题** | Agent 怎么才能越来越强？ | 怎么让 Agent 安全可靠地执行任务？ |
| **设计哲学** | 成长优先（自进化） | 安全优先（默认安全） |
| **语言** | Python + Rust CLI | Node.js / TypeScript |
| **Stars** | 13.7 万 | 24.7 万 |
| **技能/插件** | Agent 自己创建 Markdown 技能 | Plugin SDK，manifest-first |
| **记忆系统** | 三层：内置 + 外部 Provider + 会话搜索 | 单插件槽位，可替换 |
| **安全模型** | 智能审批（辅助模型判断风险） | 10+ 安全模块，默认安全 |
| **执行环境** | 6 种后端（含无服务器） | 3 种后端（偏安全沙箱） |
| **国内平台** | 飞书/钉钉/企业微信/微信 原生支持 | 飞书/QQ 扩展支持 |
| **研究能力** | 内置 RL 训练工具链 | 纯产品，无训练能力 |
| **安装复杂度** | 中等 | 低 |
| **适合** | 深度 AI 用户、追求自进化 | 轻量入门、首次尝试 Agent |

---

## 深度对比：五个维度

### 一、设计哲学

**OpenClaw — 安全优先**：
- 默认安全，高权限需显式声明
- 10+ 安全审计模块
- Docker 沙箱隔离
- 多层审批机制

**Hermes — 成长优先**：
- 从经验中学习，越用越强
- 自动提炼 Skill 的学习闭环
- 智能审批（辅助模型判断风险）
- 灵活的执行环境切换

### 二、学习闭环 vs 预定义技能

| 维度 | OpenClaw | Hermes |
|---|---|---|
| 创建方式 | 人工/开发者编写 | Agent 自动提取 + 人工可编辑 |
| 更新机制 | 人工维护 | Agent 自主判断是否更新 |
| 存储形式 | 结构化定义 | Markdown 文件 |
| 技能数量 | 5700+ 社区 Skill | 40+ 社区 + 自生成 |

### 三、记忆系统

| 维度 | Hermes | OpenClaw |
|---|---|---|
| 架构 | 三层分离（会话/持久/技能） | 单插件槽位，可替换 |
| 外部记忆 | 8 个 Provider | 可替换插件 |
| 会话搜索 | FTS5 + LLM 摘要 | 丢给主模型判断 |
| 记忆注入 | 会话启动时冻结快照 | 按需 |

### 四、子 Agent 系统

| 维度 | Hermes | OpenClaw |
|---|---|---|
| 协议 | Delegate Tool（内置） | ACP（Agent Client Protocol） |
| 最大深度 | 1 层（父→子） | 可配置 |
| 并发上限 | 3 个 | 取决于实现 |
| 防套娃 | 禁用 delegate_task | HTTP 端口控制 |

### 五、国内生态

Hermes 在中国生态上优势明显：
- 飞书、钉钉、企业微信**原生支持**
- 微信通过腾讯官方 iLink Bot API
- 支持 4000+ 模型（含智谱、月之暗面、MiniMax）

OpenClaw 通过扩展插件支持飞书和 QQ，但缺钉钉和企业微信的原生支持。

## 选择决策树

```
你的首要需求是什么？
├── 安全合规是硬要求 → OpenClaw
├── 想让 Agent 自己学习改进 → Hermes
├── 消息平台覆盖最广 → OpenClaw（25+ 渠道）
├── 主力用飞书/钉钉/企业微信 → Hermes（三者原生）
├── TypeScript 技术栈 → OpenClaw
├── Python 技术栈 → Hermes
├── 企业环境/安全合规 → OpenClaw
├── 快速探索/个人使用 → Hermes
├── 做 RL 训练研究 → Hermes（唯一有工具链）
└── 想低成本 24 小时跑 → Hermes（无服务器后端）
```

---

## 一句话总结

- **OpenClaw**：想做生产级平台——安全、稳定、可扩展。适合企业环境和第一次尝试 Agent 的用户。
- **Hermes**：非常迎合个人用户——灵活、自进化、多平台。适合深度使用和探索 Agent 能力边界的用户。

> ⚠️ **现实判断**：目前 Agent 整体还不成熟，两者都偏探索性质。不必纠结于「选哪个」，可以两个都装，根据任务场景切换。

---

## 组合策略

与编程工具的组合类似：

```
Hermes：日常助手（飞书/微信交互，自动学习）
  +
OpenClaw：安全任务（企业环境，需要严格审批）
```

---

## 本章小结

1. 两个 Agent 框架的根本差异：安全 vs 成长
2. Hermes 适合个人探索，OpenClaw 适合企业环境
3. 国内用户（用飞书/钉钉/微信）优先考虑 Hermes
4. 安全合规优先考虑 OpenClaw
5. Agent 整体还不成熟，两者可以都试试
