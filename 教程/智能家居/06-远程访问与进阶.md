# 智能家居 06 — 远程访问与进阶

> 最后一步：让你出门在外也能控制家里的一切。附完整的踩坑清单和进阶方向。

---

## 远程访问：为什么需要？

HA 部署在家庭局域网内的 NAS 上，默认只能在 **同一个 WiFi 下** 访问。但真正的智能家居需要随时随地控制：

- 出门忘记关空调？远程关
- 快递到门口？远程看门铃
- 回家路上？提前开空调和灯

实现远程访问有几种方式，这里推荐国内最实用的方案——**花生壳内网穿透**。

---

## 方案一：花生壳内网穿透（推荐）

### 第一步：安装花生壳

1. 下载花生壳客户端（支持 NAS / Windows / macOS）
2. 注册登录花生壳账号

### 第二步：配置内网穿透

1. 花生壳客户端 → 高级设置 → 内网穿透 → 新增映射
2. 填写配置：

| 参数 | 值 | 说明 |
|------|-----|------|
| 内网 IP | NAS 的局域网 IP（如 192.168.1.100） | 你的 NAS 地址 |
| 内网端口 | 8123 | HA 的默认端口 |
| 外网域名 | 花生壳生成的域名 | 如 `xxx.qicp.vip` |
| 外网端口 | 自动分配 | 花生壳随机分配 |

3. 保存 → 启动映射

### 第三步：测试

用手机流量（断开 WiFi）访问花生壳生成的外网地址，如：

```
http://xxx.qicp.vip:12345
```

如果能打开 HA 登录页面，配置成功。

---

## 手机 App 配置（内外网自动切换）

Home Assistant 官方提供了手机 App（iOS/Android），支持**双地址自动切换**：

### 配置方法

1. 下载 Home Assistant App（App Store / Google Play）
2. 手动输入地址：

| 地址类型 | 值 | 使用场景 |
|----------|-----|----------|
| **内网地址** | `http://192.168.1.100:8123` | 在家，连接 WiFi 时 |
| **外网地址** | `http://xxx.qicp.vip:12345` | 外出，使用移动网络时 |

3. App 会自动检测当前网络，在家用内网（速度快），外出用外网（花生壳穿透）。

> ⚠️ iOS 用户如果安装了 HA App，可能还需要在「家庭」App 中配置 HomeKit 中枢（如 HomePod 或 Apple TV）才能实现 HomeKit 远程访问。

---

## 方案二：HomeKit 远程访问（Apple 用户）

如果你已经通过 HomeKit Bridge 将 HA 设备桥接到了 Apple Home，并且家里有 **HomePod / Apple TV / iPad（设为家庭中枢）**，那么 Apple 会自动处理远程访问。

**优势**：无需花生壳，无需额外配置，苹果帮你穿墙。

**限制**：
- 需要 Apple 家庭中枢设备
- 只能通过「家庭」App 或 Siri 控制（无法访问 HA 完整 Web UI）
- 部分复杂设备类型可能不完全兼容 HomeKit

**建议**：花生壳 + HomeKit 双方案并存。花生壳用于完整 HA 管理，HomeKit 用于日常 Siri 控制。

## HA 原生自动化：与 AI 自动化的分工

教程第五章讲的是 AI 自动化（自然语言 + 记忆系统），但 HA 自带的自动化引擎也有不可替代的价值。

### 分工建议

| 场景类型 | 推荐方案 | 原因 |
|----------|----------|------|
| 定时开关（每天 23:00 关灯） | **HA 原生自动化** | 简单、确定、低延迟 |
| 传感器触发（门开了 → 开玄关灯） | **HA 原生自动化** | 需要亚秒级响应 |
| 安防告警（烟雾报警 → 全屋灯闪） | **HA 原生自动化** | 可靠性最高 |
| 模糊意图（「我要睡了」） | **AI Agent** | AI 理解个性化意图 |
| 复杂条件（温度 + 人数 + 时段） | **AI Agent** | 多条件组合判断 |
| 临时场景（「把客厅调成电影模式」） | **AI Agent** | 描述意图即可，无需预设 |

### HA 自动化示例（YAML）

```yaml
# configuration.yaml 或 automations.yaml
alias: 晚上关灯
trigger:
  - platform: time
    at: "23:00:00"
condition:
  - condition: state
    entity_id: binary_sensor.home_mode
    state: "on"  # 仅在家模式下执行
action:
  - service: light.turn_off
    target:
      entity_id:
        - light.living_room
        - light.dining_room
        - light.kitchen
  - service: light.turn_on
    target:
      entity_id: light.bedroom_nightlight
    data:
      brightness_pct: 10
```

## 完整踩坑清单

这里是社区用户在实际部署中遇到的常见问题及解决方案。

### 部署阶段

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Docker 镜像下载失败 | 国内网络限制 | 添加国内镜像仓库 `docker.1ms.run` |
| 容器启动后秒退 | 文件夹映射错误或不存在 | 确认 `/docker/homeassistant` 已创建 |
| 8123 端口打不开 | 网络模式未设 host 或防火墙拦截 | 检查 Container Manager 网络设置 |
| HACS 安装后不显示 | 未重启容器或多层嵌套目录 | 重启容器；检查 hacs 文件夹结构 |

### 品牌集成阶段

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 米家设备不显示 | 账号区域或设备离线 | 确认设备在米家 APP 中正常在线 |
| 海尔 Token 抓不到 | 小程序登录时机不对 | 先开小黄鸟调试 → 再登录小程序 |
| colmo 设备搜不到 | 未导入美的美居 APP | 先在美的美居 APP 中绑定，再通过 HA 集成 |
| 卡萨帝设备不识别 | 未导入海尔智家 APP | 先在海尔智家 APP 中绑定，再通过 HA 集成 |
| 集成安装后提示「未加载」 | 缺少依赖或版本不兼容 | 查看 HA 日志 → 设置 → 系统 → 日志 |

### AI 集成阶段

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Token 不生效 | 未重启 Hermes | toolset 在启动时加载，重启生效 |
| Hermes 连不上 HA | 网络不通 | 确认同一局域网；防火墙开放 8123 |
| 设备名匹配失败 | 实体 ID 与自然语言不匹配 | 先用「列出所有设备」获取准确实体 ID |
| Cron 不执行 | CLI 模式退出 | 切换到 gateway 模式 |

### 远程访问阶段

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 花生壳外网打不开 | 映射未启动或端口冲突 | 检查花生壳穿透状态；尝试更换外网端口 |
| 手机 App 连不上 | 地址配置错误 | 检查内网/外网地址是否正确 |
| 外网访问很慢 | 花生壳免费版带宽限制 | 升级付费版或切换其他穿透方案 |

## 进阶方向

恭喜你完成了基础部署！以下是进阶探索方向：

### 1. 更多品牌接入

HA 2700+ 集成远不止这五个品牌：
- **Sonos**：全屋音响
- **Philips Hue**：智能灯光
- **Aqara**：传感器 + 门锁
- **BroadLink**：红外遥控（控制传统家电）
- **ESPHome**：DIY 传感器（温湿度、PM2.5 等）

### 2. 高级自动化

- **Blueprint**：社区共享的自动化模板，一键导入
- **Node-RED**：可视化拖拽式自动化编排（比 YAML 友好）
- **Presence Detection**：基于手机 WiFi/蓝牙判断是否在家

### 3. Dashboard 美化

- **Mushroom Cards**：现代风格的 UI 卡片
- **Minimalist**：极简主义仪表盘
- **Floorplan**：基于房屋平面图的交互式控制面板

### 4. AI 深度整合

- **本地 LLM**：Ollama + HA，完全离线 AI 控制
- **多 Agent 协作**：Hermes + Claude Code，不同 Agent 分管不同场景
- **语音唤醒**：接入本地唤醒词（如 Espressif 的 ESP-SR）

### 5. 能源管理

- **Energy Dashboard**：HA 内置能源面板
- 接入智能电表、太阳能逆变器、电动车充电桩
