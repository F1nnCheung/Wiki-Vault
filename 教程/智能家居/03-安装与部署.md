# 智能家居 03 — 安装与部署（群晖 NAS + Docker）

> 以群晖 NAS + Docker 为主力方案，一步步完成 Home Assistant 的安装。其他 NAS 品牌（威联通、绿联、极空间等）流程类似。

---

## 环境准备

### 第一步：安装 Container Manager（Docker 环境）

群晖套件中心 → 搜索 **Container Manager** → 安装。

![Container Manager 是群晖的 Docker 管理工具]

### 第二步：配置国内镜像仓库

由于默认 Docker Hub 在国内访问困难，需要添加国内镜像加速：

1. 打开 Container Manager → 注册表 → 设置
2. 添加镜像仓库地址：`https://docker.1ms.run`
3. 勾选启用

添加后应该能看到各种镜像名称正常显示，说明连接成功。

### 第三步：创建共享文件夹

1. 控制面板 → 共享文件夹 → 新增
2. 名称：`docker`
3. 位置：默认
4. 创建后，在 File Station 中确认 `/docker` 目录存在

这个文件夹将用于持久化 Home Assistant 的所有配置数据。

---

## 安装 Home Assistant

### 第一步：下载镜像

Container Manager → 注册表 → 搜索 `home-assistant` → 下载。

> 镜像较大，下载需要几分钟，取决于网络速度。

### 第二步：创建容器

容器 → 新增 → 选择 `home-assistant` 镜像 → 设置以下参数：

**容器名称**：`homeassistant`（或任意名称）

**文件夹映射**（关键！）：
| 主机路径 | 挂载路径 | 说明 |
|----------|----------|------|
| `/docker/homeassistant` | `/config` | HA 所有配置持久化在这里 |

> ⚠️ 先确认 `/docker/homeassistant` 文件夹已在 File Station 中创建好。

**环境变量**：
| 变量名 | 值 | 说明 |
|--------|-----|------|
| `TZ` | `Asia/Shanghai` | 时区设置 |

**网络模式**：选择 **host**（让 HA 直接使用 NAS 的网络，避免端口映射问题）

### 第三步：启动容器

点击「应用」→ 回到 Container Manager → 选中 `homeassistant` 容器 → 操作 → 启动。

第一次启动需要几分钟初始化。状态变为「运行中」后继续下一步。

### 第四步：访问 Home Assistant

浏览器打开：`http://[NAS_IP]:8123`

> 群晖默认端口是 5000，HA 默认端口是 8123。
>
> 例如：群晖地址是 `http://192.168.1.100:5000`，HA 的地址就是 `http://192.168.1.100:8123`

首次进入会看到初始化向导：

1. 创建你的家庭名称（如「我的家」）
2. 设置用户名和密码（**请牢记，这是管理员账号**）
3. 选择位置（用于自动化中的日出日落、天气等）

完成后进入 HA 主界面。此时还没有接入任何设备，界面会比较空——接下来安装 HACS 商店后，就能开始接入品牌设备了。

---

## 安装 HACS 插件商店

HACS（Home Assistant Community Store）是接入第三方品牌集成的必经之路。由于国内网络环境，推荐使用**离线安装**方式。

### 第一步：创建文件夹

File Station → 进入 `/docker/homeassistant` → 新建文件夹：

```
custom_components/
└── hacs/
```

> ⚠️ 路径要精确。如果已有 `custom_components` 文件夹，在其下新建 `hacs` 即可。

### 第二步：下载 HACS 安装包

前往 [HACS GitHub Releases](https://github.com/hacs/integration/releases) 下载最新版 `hacs.zip`。

解压后将**所有文件**放入 `/docker/homeassistant/custom_components/hacs/` 文件夹。

> ⚠️ 注意不要多套一层 hacs 目录！解压后应该是 `hacs/__init__.py` 等文件直接在 hacs 文件夹下。

### 第三步：重启容器

Container Manager → 选中 `homeassistant` → 操作 → 重启。

> 容器重启最彻底，比在 HA Web UI 中「重新启动」更可靠。

### 第四步：添加 HACS 集成

1. HA Web UI → 设置 → 设备与服务 → 右下角「+ 添加集成」
2. 搜索 **HACS** → 点击安装
3. 全部勾选 → 下一步
4. 会跳转到 GitHub 授权页面

> GitHub 可能打不开？尝试：
> - 电脑连接手机热点
> - 使用科学上网工具
> - 安装游戏加速器

5. 输入 GitHub 授权码 → 完成

安装成功后，侧边栏会出现 **HACS** 图标。这就是你的「HA 应用商店」。

## 简单检查清单

在进入下一章之前，确认以下事项：

- [ ] NAS IP:8123 可以访问 HA Web UI
- [ ] 已创建管理员账号密码
- [ ] HACS 已成功安装，侧边栏可见
- [ ] 容器状态为「运行中」，且设置了自动重启

---

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 镜像下载失败 | 检查镜像仓库地址是否添加并勾选启用 |
| 容器启动后立即停止 | 检查文件夹映射是否正确，`/docker/homeassistant` 是否存在 |
| 8123 端口无法访问 | 检查网络模式是否设为 host；NAS 防火墙是否开放 8123 |
| HACS 搜不到 | 确认重启的是容器而非仅重启 HA Core；检查 hacs 文件夹没有多嵌套一层 |
| GitHub 授权打不开 | 尝试手机热点；或在路由器层面配置代理 |
