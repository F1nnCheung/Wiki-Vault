# 智能家居 05 — AI 语音控制与自动化

> 本章是教程的「灵魂」。接入 AI Agent（Hermes）后，智能家居的交互方式发生质变——从「手动点 APP」到「说一句话控制全家」，再到「AI 记住你的习惯并自动执行」。

---

## 为什么需要 AI 控制？

HA 本身已经很强大了：统一面板、自动化规则、定时任务。但交互方式仍然是「手动操作」：

```
传统方式：
  开灯 → 打开 HA APP → 找到客厅灯 → 点击开关
  调温度 → 打开 APP → 找到空调 → 拖温度条

AI 方式：
  「把客厅灯调到 30%，色温暖白」
  「空调设到 24 度，制冷模式」
  「我要睡了」→ AI 自动执行你的完整睡前场景
```

**AI Agent 的价值不在于「替代 HA 的自动化引擎」，而在于「降低交互门槛」和「理解模糊意图」。**

---

## 前提条件

| 条件 | 说明 |
|------|------|
| Home Assistant 已安装运行 | 任何安装方式（Docker/NAS/树莓派）均可 |
| Long-Lived Access Token | HA 用户配置中生成 |
| Hermes Agent 已安装 | v0.13+ 推荐 |
| 网络互通 | Hermes 能访问 HA 的 IP:8123 |

> 如果你还没有安装 Hermes Agent，请参考 [[Wiki/wiki/topics/hermes-agent-guide|Hermes Agent 完整教程]]。

## 配置步骤

### 第一步：获取 HA Long-Lived Access Token

1. 打开 Home Assistant Web UI
2. 点击左下角用户头像
3. 滚动到底部 → 「Long-Lived Access Tokens」
4. 创建 token，命名如 `hermes-control`
5. **复制保存，只显示一次！**

### 第二步：配置 Hermes

编辑 `~/.hermes/.env`：

```bash
HASS_TOKEN=your-long-lived-access-token
HASS_URL=http://your-homeassistant-ip:8123
```

**仅需这两个变量。** Hermes 检测到 `HASS_TOKEN` 后，启动时自动加载 `homeassistant` toolset。

### 第三步：验证连接

启动 Hermes，在对话中输入：

```
列出我家所有的设备
```

如果连接成功，Hermes 会返回 HA 中的所有实体列表。这说明你的自然语言已经能「看见」家里的所有设备了。

> ⚠️ **`HASS_TOKEN` 修改后必须重启 Hermes**，toolset 在启动时加载。

## 五大实战场景

### 场景一：自然语言控制

不再需要记住实体 ID。用日常语言描述即可：

```
把客厅灯调到 30% 亮度，色温暖白
关掉所有卧室的灯
把空调设到 24 度，制冷模式
```

AI 自动将自然语言映射到正确的设备和服务调用。说错一点也没关系，AI 会模糊匹配。

### 场景二：复杂条件逻辑

```
如果现在室外温度超过 30 度，打开客厅空调设到 25 度。
如果低于 30 度，只开风扇。
```

Hermes 的执行流程：
1. 查询温度传感器（`sensor.outdoor_temp`）
2. 判断数值 → 30°C 以上？
3. 是 → 调用 HA API 开空调 25°C
4. 否 → 调用 HA API 开风扇

**一句话完成「感知→判断→执行」全流程。**

### 场景三：定时自动化（Cron）

```
设一个定时任务：每天晚上 11 点，关掉所有灯，只留卧室小夜灯。
```

Hermes 等价于在内部执行：

```bash
hermes cron create "0 23 * * *"
# prompt: 关掉所有灯，只留卧室小夜灯
```

> ⚠️ **Cron 任务在 gateway 模式下才持久。** 纯 CLI 模式退出后 cron 不执行。

### 场景四：Telegram 远程控制

配置 Hermes Telegram 渠道后，在路上发消息：

```
我快到家了，提前开空调和玄关灯
```

Hermes 收到 Telegram 消息 → 调用 HA API → 设备开启。

**人在路上，家已就绪。**

### 场景五：Philips Hue 场景控制

如果你有 Philips Hue 灯具（通过 OpenHue Skill）：

```
把书房切换到"专注"场景
把客厅灯设成日落色，亮度 60%
```

## 核心特性：记忆系统

这是 AI 智能家居**区别于传统自动化的关键能力**。

### 你只需要告诉 AI 一次

```
记住：我喜欢睡觉时客厅全关，卧室留小夜灯 10% 亮度，空调 26 度
```

### 以后只需要说三个字

```
我要睡了
```

Hermes 从记忆中读取你的偏好 → 自动执行完整场景。

**从「手动配置自动化规则」进化到「AI 记住习惯并自动执行」。**

```
传统 HA 自动化：
  触发器：23:00
  条件：无
  动作：关客厅灯 + 开小夜灯 10% + 空调 26°C
  （需要手动逐条配置 YAML）

AI 记忆：
  「记住我喜欢睡觉时…」
  → 「我要睡了」
  （零配置）
```


## 社区案例：LLM 接管门铃对讲

一位 HA 用户的实验性方案：当门铃响且检测到家里没人时，让本地 LLM 接管门铃的对讲功能。

### 流程

```
1. 门铃响 → HA 检测到无人在家
2. 触发 Hermes Agent
3. Hermes 通过 TTS 对门铃说话：
   "你好，主人不在家，请问你是谁？有什么事？"
4. 门铃录音 → STT 转文字 → Hermes 理解内容
5. Hermes 根据情况回应：
   快递 → "请放门口"
   朋友 → "我帮你转告"
6. 同时发 Telegram 通知主人
```

### 关键点

- **全程使用本地模型（Ollama）**，不依赖云端 API
- **隐私完全可控**
- 这不是科幻，是 2026 年已经在跑的方案

参考来源：[Yahoo Tech: Let Local LLM Control Video Doorbell](https://tech.yahoo.com/home/articles/let-local-llm-control-video-153016480.html)

## 能力边界

| ✅ AI 擅长 | ❌ AI 不适合 |
|-----------|-------------|
| 自然语言控制现有设备 | 添加新设备（需 HA UI 操作） |
| 条件判断 + 多步骤自动化 | 复杂自动化（建议用 HA 原生 Automation） |
| 定时任务 + 远程控制 | 需要亚秒级响应的场景 |
| 记住偏好，简化交互 | 离线环境（AI 模型需要推理） |

> 最佳实践：**HA 原生自动化负责「高频、确定、低延迟」的场景；AI 负责「低频、模糊、个性化」的场景。两者互补而非替代。**

## 踩坑提醒

| 问题 | 解决方案 |
|------|----------|
| Token 设置后不生效 | **重启 Hermes**，toolset 启动时加载 |
| Hermes 无法访问 HA | 检查网络互通（同一个局域网/VPN） |
| 实体名匹配失败 | 尝试使用 HA 中的精确实体 ID 替代自然语言名称 |
| Cron 不执行 | CLI 模式下 cron 不持久，切换到 gateway 模式 |
| HA 在远程服务器上 | 确保 `HASS_URL` 可达，必要时配置内网穿透 |
