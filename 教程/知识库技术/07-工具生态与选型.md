## 四大实战场景

### 场景一：个人知识管理

**需求**：管理文章、笔记、想法，能自然语言搜索，知识持续积累。

**推荐方案**：LLM Wiki + Obsidian

```
Obsidian Vault（Markdown 笔记库）
  ├── raw/（Web Clipper 保存文章）
  ├── wiki/（Claude Code / Copilot 自动整理）
  └── CLAUDE.md（规则配置）
```

**为什么不是 RAG**：
- 个人笔记 < 500 篇，不需要向量数据库
- LLM Wiki 的知识复利远超 RAG 的"每次重新搜索"
- Git 管理版本天然适合

**进阶**：加入 ChromaDB + Ollama 做 RAG 辅助检索（详见 Hermes RAG 方案）。

### 场景二：企业知识库

**需求**：内部文档问答，需要高准确率、引用溯源、权限控制。

**推荐方案**：Classic RAG + Milvus + 重排

```
文档 → 语义分块 → BGE-M3 Embedding → Milvus（HNSW 索引）
查询 → 查询重写 → 混合检索 → 重排（Cross-Encoder）→ 引用溯源
```

**关键优化**（投入产出比排序）：
1. Prompt 护栏："不知道就说不知道"
2. 引用溯源：要求标注来源段落
3. 查询重写：把口语转为检索语言
4. 重排：Cross-Encoder 对 top 20 重排序
5. 元数据过滤：按部门/日期/类型预筛选

### 场景三：代码库理解

**需求**：让 AI 编程助手真正理解项目结构，不只是读文件。

**推荐方案**：GitNexus（或 Graphify）+ Claude Code / Codex MCP 集成

```
仓库 → GitNexus（浏览器端 AST 解析）→ 代码知识图谱 → MCP 工具
AI 编程 → 改代码前查图谱 → 了解影响面 → 减少"改一处炸一片"
```

**6 阶段索引管道**：
1. 结构阶段：遍历文件树
2. 解析阶段：AST 提取函数/类/方法
3. 关联阶段：跨文件调用链/继承/类型依赖
4. 聚类阶段：Leiden 算法自动分组
5. 流程阶段：追踪执行流
6. 搜索阶段：BM25 + 语义 + RRF 融合

### 场景四：智能家居 + RAG

**需求**：用自然语言控制全家设备，记住用户偏好。

**推荐方案**：Home Assistant + Hermes Agent + ChromaDB

```
Home Assistant（设备中控）
  ↓ MCP / Long-Lived Token
Hermes Agent（AI 大脑）
  ↓ Ollama Embedding + ChromaDB
记忆系统（偏好/场景/历史）
```

## 选型决策树

```
START
  │
  ├─ 个人使用，< 500 篇资料？
  │   └─ → LLM Wiki（Markdown + LLM Agent）
  │
  ├─ 企业知识库，需要高准确率？
  │   └─ → Classic RAG + Milvus + 重排
  │
  ├─ 答案依赖关系和链条？
  │   └─ → Classic RAG + Graph RAG（双检索）
  │
  ├─ 问题复杂，一开始不知道查什么？
  │   └─ → Agentic RAG
  │
  ├─ 需要 AI 理解代码项目结构？
  │   └─ → GitNexus / Graphify（代码知识图谱）
  │
  ├─ 数据不能出机器？
  │   └─ → Ollama + ChromaDB（全本地）
  │
  └─ 预算敏感？
      └─ → ChromaDB + bge-m3 + 开源 LLM
```

## 投入产出比排名

从零搭建知识库，最值得投入的方向：

| 排名 | 投入 | 产出 | 适合谁 |
|------|------|------|--------|
| 🥇 | **LLM Wiki** | 知识开始复利积累 | 所有人 |
| 🥈 | **Prompt 护栏** | 幻觉大幅减少 | RAG 用户 |
| 🥉 | **语义分块** | 检索质量质的飞跃 | RAG 用户 |
| 4 | **查询重写** | 缩小语义鸿沟 | RAG 用户 |
| 5 | **混合检索** | 关键词+语义互补 | RAG 用户 |
| 6 | **重排（Rerank）** | 排序质量飞跃 | RAG 用户 |
| 7 | **知识图谱** | 关系链查询 | 企业用户 |
| 8 | **Agentic RAG** | 自主推理 | 复杂场景 |

## 完整学习路线图

```
🟢 入门（1-2 天）
├── 阅读 01 总览 → 建立全局认知
├── 阅读 02 RAG 架构 → 理解三种检索范式
└── 搭一个 ChromaDB 原型 → 感受"能跑"
    产出：一个能跑但粗糙的 RAG 原型

🟡 进阶（3-5 天）
├── 阅读 03 RAG 优化 → 系统性优化原型
├── 阅读 04 知识图谱 → 理解关系建模
├── 阅读 05 LLM Wiki → 理解编译器范式
└── 阅读 06 向量数据库 → 懂得选型
    产出：优化到"能用"的 RAG + 一个小型知识图谱

🔴 深入（1-2 周）
├── 阅读 07 工具与选型 → 确定自己场景的最佳方案
├── 搭建 LLM Wiki → 开始知识复利积累
├── 安装 GitNexus → 生成项目代码图谱
└── 探索 Agentic RAG → 需要时引入自主推理
    产出：一个持续运行的 LLM Wiki + 代码知识图谱
```
