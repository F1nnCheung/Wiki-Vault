# 2.5.1 Codex 介绍与安装

> Codex 是 OpenAI 的「无人值守」编程 Agent——给它一个任务描述，它从需求分析到部署 PR 全流程自动化。适合批量任务和自动化脚本，但不适合日常编码。

---

## 概述

Codex 是 OpenAI 推出的云端 AI 编程 Agent，集成于 ChatGPT。与 Claude Code 或 Cursor 不同，Codex 在**云端独立运行**——你不需要保持 IDE 打开，它自己能完成从分析到提交的全流程。

**入口**：chatgpt.com → 切换到 Codex 模式

---

## 核心特点

| 特点 | 说明 |
|---|---|
| **云端运行** | 不占用本地资源，关闭电脑也能继续 |
| **全流程自动化** | 需求分析 → 代码编写 → 测试 → 部署 PR |
| **Git 集成** | 克隆仓库、创建分支、提交 PR |
| **长期任务** | 大型重构和迁移可以跑几小时 |

---

## Codex vs 其他工具

| 维度 | Codex | Claude Code | Cursor |
|---|---|---|---|
| 运行位置 | 云端 | 本地终端 | 本地 IDE |
| 交互方式 | 下任务 → 等结果 | 持续对话 | IDE 内置 |
| 自动化程度 | ⭐⭐⭐⭐⭐（最高） | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 代码补全 | ❌ 没有 | ❌ 没有 | ✅ 实时 |
| 适合 | 批量任务 | 深度推理 | 日常编码 |

> 💡 **定位**：Codex 不是日常编码工具，而是「当你有一堆重复工作要做时，丢给它然后去喝咖啡」的自动化工具。

---

## 安装与配置

### 方法一：Web 界面（推荐）

Codex 是云端服务，最简单的入口是 ChatGPT 网页版：

1. 访问 **chatgpt.com**
2. 登录 OpenAI 账号（需要 Plus/Pro/Team 订阅）
3. 切换到 **Codex 模式**
4. 连接 GitHub 账号（授权仓库访问权限）

### 方法二：CLI 工具（可选）

如果需要从终端操作：

```bash
# 安装 Codex CLI
npm install -g @openai/codex

# 登录 OpenAI 账号
codex login

# 在当前项目目录初始化
codex init
```

### 权限配置

Codex 需要以下权限：
- **GitHub 仓库读写**：用于克隆、创建分支、提交 PR
- **NPM/Pip 等包管理器**：用于安装依赖
- **终端执行**：用于运行测试和构建

> ⚠️ 建议：给 Codex 一个**独立的 GitHub 仓库权限**（按仓库授权，不要给全账号权限）。

## 定价

Codex 按 Token 消耗计费，包含在 ChatGPT 订阅中：

| 计划 | 月费 | Codex 使用额度 |
|---|---|---|
| Plus | $20 | 有限额度 |
| Pro | $200 | 较高额度 |
| Team | $25/人 | 团队共享额度 |

> ⚠️ Codex 的全流程自动化会消耗大量 Token。一个中等复杂度的任务可能消耗 $5-20 不等的 Token 费用。建议先用小任务测试，了解消耗水平。

---

## 本章小结

1. Codex = 云端「无人值守」编程 Agent
2. 不需要本地安装，通过 ChatGPT Web 界面使用
3. 核心优势：全流程自动化，关闭电脑也能跑
4. 核心短板：没有实时代码补全，Token 消耗大
5. 定价按量消耗，中等任务 $5-20/次
