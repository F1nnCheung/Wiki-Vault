# 2.2.2 Claude Code 安装与配置

> 两条安装路线 + 国内用户第三方模型配置 + 进阶配置工具。本章从零带你完成 Claude Code 的安装和基础配置。

---

## 安装前准备

**前置要求**：
- Node.js 18+（推荐使用 nvm 管理版本）
- Git（用于版本控制和仓库操作）
- 终端（macOS 用 Terminal/iTerm2，Windows 用 WSL2）

**验证前置**：
```bash
node --version  # 应 >= v18.0.0
git --version   # 应 >= 2.0.0
```

---

## 路线一：CLI 安装（推荐开发者）

这是最主流、功能最完整的安装方式：

```bash
# 方法 A：官方脚本安装（推荐）
curl -fsSL https://claude.ai/install.sh | bash

# 方法 B：npm 全局安装
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

**启动**：
```bash
cd your-project    # 进入你的项目目录
claude             # 启动 Claude Code
```

首次启动会引导你完成 OAuth 认证（需要 Anthropic 账号和 API 订阅）。

---

## 路线二：桌面客户端（适合非技术用户）

如果你不习惯命令行，可以使用桌面客户端：

1. 从 **claude.ai/download** 下载 Claude Desktop Client
2. 安装后打开，进入 **Help → Troubleshooting → Enable Developer Mode**
3. 切换到 **Code 模式**（`</>` 图标）

桌面客户端提供三种模式：
- **Chat**：普通对话
- **Cowork**：桌面 Agent，可操作桌面应用
- **Code**：GUI 版 Claude Code（功能等同 CLI）

> ⚠️ 桌面客户端的 Code 模式功能上等价于 CLI，但扩展性（MCP/Skills 配置）不如 CLI 灵活。

---

## 国内用户必读：配置第三方模型

> ⚠️ **重要**：由于 Anthropic 账号容易封禁，国内用户强烈建议配置第三方模型供应商，无需 Anthropic 账号。

### 方法一：settings.json 配置（推荐）

配置文件位置：`~/.claude/settings.json`

```json
{
  "hasCompletedOnboarding": true,
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的API Key",
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_MODEL": "deepseek-chat"
  }
}
```

只需替换 API Key 和模型名即可。各供应商的配置参数见下方。

### 方法二：桌面客户端 3P 模式

1. 启用 Developer Mode
2. Developer → Configure Third-Party Inference
3. 配置 Gateway：填入 API 地址和密钥
4. **不需要 Anthropic 账号**、不需手机验证、不需订阅

## 配置的三个层次

理解这三个层次，才能做好 Claude Code 的配置：

| 层次 | 文件 | 用途 | 优先级 |
|---|---|---|---|
| 工具层 | `settings.json` | 权限、模型、MCP 配置 | 基础设施 |
| 规则层 | `CLAUDE.md` | 项目规范、编码约定 | 最重要 |
| 学习层 | auto memory | AI 自动记录的偏好和经验 | 润物细无声 |

### 规则层详解：CLAUDE.md

`CLAUDE.md` 是 Claude Code 的「长期项目记忆」。每次会话启动时自动加载。

```markdown
# 项目规范
- 技术栈：Next.js 14 + TypeScript + Prisma + Tailwind
- 使用 Server Components 优先，避免不必要的 'use client'
- 所有 API 路由统一放 src/app/api/，命名遵循 RESTful 规范
- 测试用 Vitest，覆盖率 >80%
- 禁止直接修改 .env 文件
```

**CLAUDE.md 写作原则**：
1. **控制在 200 行以内**——太长会消耗大量 Token
2. **聚焦核心规范**，详细文档放外部引用
3. **写约束，不写教程**——AI 需要知道「边界」，不需要被教育
4. **每层职责分明**——见下方层级结构

> 📖 **推荐阅读**：[[Wiki/wiki/topics/claude-md-12-rules|CLAUDE.md 12 条规则深度解析]]——Karpathy 原版 4 条 + 辰北 8 条增量规则，每条都对应真实翻车现场。实验数据显示失误率从 41% 降至 3%。文末附可复制的完整 12 条模板。

#### CLAUDE.md 的层级结构

CLAUDE.md 是**分层的**，Claude Code 从多个位置发现并加载，从最全局到最具体：

| 层级 | 位置 | 作用 | 加载方式 |
|---|---|---|---|
| **用户全局** | `~/.claude/CLAUDE.md` | 所有项目共享的个人偏好 | 启动时 |
| **项目根** | `./CLAUDE.md` | 团队共享，检入 Git | 启动时 |
| **本地覆盖** | `./CLAUDE.local.md` | 个人偏好，Git ignored | 启动时 |
| **子目录** | `packages/web/CLAUDE.md` | Monorepo 模块专属 | **按需** |
| **规则文件** | `.claude/rules/*.md` | 路径作用域（通过 `globs`） | 匹配时 |

> 🔑 **核心机制：拼接而非覆盖**。所有 CLAUDE.md 文件会被拼接成一个 prompt 喂给模型——没有规则会被删除或替换，只是被放在一起。因此**不要在不同层级重复同一规则**来「加强」它——冲突是未定义行为。越靠近工作目录越优先，但不保证确定性。

**实践建议**：
- 全局 `~/.claude/CLAUDE.md`：放个人偏好（缩进风格、命名惯例）
- 项目根 `./CLAUDE.md`：放团队约定（技术栈、测试命令、架构约束）
- 子目录 `packages/xxx/CLAUDE.md`：放模块特定规则（API 约定、DB schema 规范）
- `.claude/rules/`：用 `globs` 实现路径级精确控制

> 💡 更多细节见 [[Wiki/wiki/entities/claude-code#CLAUDE.md 层级结构|Claude Code 实体页 → CLAUDE.md 层级结构]]。

### 工具层详解：settings.json

`settings.json` 是 Claude Code 的**工具层配置中枢**，控制四件事：模型接入、权限边界、MCP 工具连接、自动化钩子。

#### 模型接入

通过 `env` 字段指定使用的模型和 API 端点。完整配置见上方「国内用户必读：配置第三方模型」一节。核心字段：

| 字段 | 作用 |
|---|---|
| `ANTHROPIC_AUTH_TOKEN` | API Key |
| `ANTHROPIC_BASE_URL` | API 端点地址 |
| `ANTHROPIC_MODEL` | 模型标识符 |

#### 权限边界

用三类规则控制 Claude Code 能做什么、不能做什么：

| 规则 | 含义 | 适用场景 |
|---|---|---|
| `allow` | 自动放行，不弹确认框 | 高频低风险操作（lint、test、git status） |
| `ask` | 弹窗询问用户确认 | 中风险操作（修改文件、安装依赖） |
| `deny` | 直接拒绝执行 | 高风险操作（操作 .env、force push、rm -rf） |

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test:*)",
      "Bash(npm run lint:*)",
      "Bash(git status:*)",
      "Bash(git diff:*)" 
    ],
    "deny": [
      "Bash(git push --force:*)",
      "Bash(rm -rf:*)",
      "Read(.env:*)"
    ]
  }
}
```

也可以用 `/permissions` 命令在会话中动态管理权限规则。

#### MCP 工具连接

通过 MCP（Model Context Protocol）将外部工具和数据源接入 Claude Code：

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-filesystem", "/path/to/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-github"]
    }
  }
}
```

配置后，Claude Code 可以读写指定目录、操作 GitHub Issues/PR、查询数据库等。MCP 是 Claude Code 扩展能力的核心机制。

#### 自动化钩子（Hooks）

在特定事件触发时自动执行脚本，实现工作流自动化：

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "command": "cat ~/.claude/CLAUDE.md | head -20"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "echo '文件已修改，建议运行测试'"
      }
    ]
  }
}
```

常用事件：`SessionStart`（会话开始）、`PreToolUse`（工具调用前）、`PostToolUse`（工具调用后）、`Stop`（会话结束）。

> 💡 **层次关系**：`settings.json` 是基础设施，它决定了 Claude Code "能调用什么工具、用什么模型、遵守什么边界"。写好后很少频繁改动。

### 学习层详解：auto memory

**auto memory** 是 Claude Code 的**长期记忆系统**——AI 在对话过程中自动记录你的偏好、习惯和经验，跨越会话持久生效。

#### 它和 CLAUDE.md 的区别

| 维度 | CLAUDE.md（规则层） | auto memory（学习层） |
|---|---|---|
| **写入方式** | 你手动编写 | AI 自动记录 |
| **内容性质** | 项目规范、编码约定、硬性约束 | 个人偏好、经验总结、软性习惯 |
| **更新频率** | 低频，重大变更时手动改 | 高频，每次对话都可能更新 |
| **生效方式** | 每次会话启动自动加载 | 随对话推进逐步注入 |
| **存储位置** | 项目根目录 `./CLAUDE.md` | `~/.claude/projects/<项目>/memory/` |

#### 如何使用

1. **自动运行**：无需手动配置。Claude Code 在对话中会自动识别值得记忆的信息（比如你说"我不喜欢用 any 类型"、"测试一定要跑 Vitest"），并记录到 memory 中
2. **手动编辑**：使用 `/memory` 命令随时查看和编辑记忆内容
3. **查看效果**：AI 在后续对话中会自动应用这些记忆，比如不再建议 any 类型、主动运行 Vitest

#### 典型记忆内容

- 「用户偏好 Server Components 优先」
- 「代码风格：单引号、分号结尾」
- 「测试框架统一用 Vitest」
- 「部署前必须检查 .env.example 同步」

> ⚠️ **注意**：auto memory 是「润物细无声」的学习层——你不需要主动管理它，但要知道它存在。如果 AI 老是犯同样的"错"，检查一下是不是 memory 里记录了对你不利的偏好，用 `/memory` 修正。

#### 三层协作关系

```
settings.json → 决定 AI「能做什么」（工具边界）
      ↓
CLAUDE.md    → 告诉 AI「该怎么做事」（项目规范）
      ↓
auto memory  → 让 AI「越来越懂你」（个人偏好）
```

三层从硬到软、从项目到个人、从手动到自动，构成完整的配置体系。

## 进阶配置工具

### CC Switch

GUI 管理多工具的 Provider/MCP/Skills 配置，支持一键热切换。

- **用途**：在不同项目/不同模型间快速切换配置
- **适用**：同时管理多个项目的开发者

### Claude Code Router

按任务类型智能路由到不同模型，降低成本 70%+。

- **用途**：简单任务自动用便宜模型，复杂任务自动切贵模型
- **适用**：Token 消耗大的重度用户

## 安装后第一步

安装完成后，按以下顺序完成初始化：

```bash
# 第一步：在项目根目录生成 CLAUDE.md
/init

# 第二步：编辑 CLAUDE.md，补充技术栈和编码规范
/memory
```

### 最小闭环验证

完成以上步骤后，做一个快速验证：

```
你：这个项目用的是什么技术栈？
  → AI 应该能准确回答（说明 CLAUDE.md 生效了）

你：帮我看看项目结构
  → AI 应该能列出文件结构（说明文件读写正常）

你：有哪些待改进的地方？
  → AI 应该能给出建议（说明理解了项目）
```

## 常见安装问题

### Q: 安装后 `claude` 命令找不到？

```bash
# 检查 npm 全局安装路径
npm list -g --depth=0

# 确保 npm 全局 bin 在 PATH 中
echo $PATH | grep "npm"
```

### Q: 认证失败？

- 检查 API Key 是否正确
- 确认账号有有效订阅（或使用第三方模型绕过）
- 检查网络是否能访问 Anthropic API

### Q: 国内网络无法访问？

- 使用第三方模型配置（见上方「国内用户必读」）
- 或使用代理工具

## 本章小结

1. CLI 安装是主流选择，桌面客户端适合非技术用户
2. 国内用户必须配置第三方模型（推荐 DeepSeek 入门，火山引擎进阶）
3. 安装后第一步：`/init` 生成 CLAUDE.md
4. 配置三个层次：settings.json（工具）→ CLAUDE.md（规则）→ auto memory（学习）
5. 进阶工具 CC Switch 和 Router 在需要时再装
