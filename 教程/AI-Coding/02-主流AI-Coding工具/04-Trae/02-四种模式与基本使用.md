# 2.4.2 Trae 四种模式与基本使用

> Trae 的四种模式覆盖了从「问答」到「一句话生成项目」的全谱系。关键是理解每种模式的适用场景——用对模式，效率翻倍。

---

## 四种模式总览

| 模式 | 快捷键 | AI 角色 | 适用场景 |
|---|---|---|---|
| **Chat** | `Cmd/Ctrl + L` | 技术顾问 | 代码解释、Bug 排查、技术咨询 |
| **Builder** | `Cmd/Ctrl + B` | 快速搭建者 | CRUD 页面、数据看板、表单 |
| **SOLO** | `Cmd/Ctrl + I` | 项目生成器 | 从零生成完整项目 |
| **IDE** | 默认 | 智能补全 | 传统编码 + AI 实时补全 |

---

## 模式一：Chat — 技术顾问

### 是什么

侧边栏 AI 对话，选中代码提问，获取分析和建议。

### 怎么用

```
1. 选中代码（或直接在 Chat 中输入）
2. Cmd/Ctrl + L
3. 输入问题
4. AI 分析并给出建议
5. 点击 Apply 直接应用修改建议
```

### 使用示例

```
✅ 代码解释：
"这段代码的逻辑是什么？为什么用了 useMemo？"

✅ Bug 排查：
"为什么这个表单提交后没有触发验证？"

✅ 重构建议：
"这个组件有什么可优化的地方？"

✅ 最佳实践：
"按照 React 最佳实践，这个 useEffect 应该怎么写？"
```

### 最佳场景

- ✅ 理解陌生代码
- ✅ 排查 Bug
- ✅ 获取优化建议
- ✅ 学习新技术
- ❌ 需要实际修改代码（用 Builder 或 Cmd+K）

---

## 模式二：Builder — 快速搭建

### 是什么

快速搭建模式——描述需求，AI 自动创建文件和代码。

### 怎么用

```
1. Cmd/Ctrl + B 打开 Builder
2. 描述需求：
   "创建一个用户管理页面，包含列表、搜索、新增按钮"
3. AI 自动创建组件、样式、路由
4. 预览效果，需要调整时继续对话
```

### 使用示例

```
✅ CRUD 页面：
"做一个文章管理后台，列表显示标题/作者/时间，支持搜索和删除"

✅ 数据看板：
"做一个销售数据看板，包含月度趋势图、Top10 产品、转化率卡片"

✅ 表单页面：
"做一个用户注册表单，包含邮箱、密码、确认密码、手机号验证"
```

### Builder vs SOLO

```
Builder：在当前项目中搭建功能模块
SOLO：从零创建完整项目
```

### 最佳场景

- ✅ 快速搭建 CRUD 页面
- ✅ 数据看板和仪表盘
- ✅ 表单页面
- ✅ 在现有项目上添加功能模块
- ❌ 需要从零创建完整项目（用 SOLO）

---

## 模式三：SOLO — 项目生成器

### 是什么

一句话生成完整项目——从前端到后端，从配置到部署脚本。

### 怎么用

```
1. Cmd/Ctrl + I 打开 SOLO
2. 描述你的项目想法：
   "做一个待办事项应用，支持添加、完成、删除，
    使用 React + TypeScript + Tailwind，数据存 localStorage"
3. AI 生成完整项目结构
4. 预览效果
5. 继续用 Chat 或 Builder 迭代修改
```

### 使用示例

```
✅ 简单应用：
"做一个番茄钟应用，25分钟工作+5分钟休息，有音效提醒"

✅ 工具网站：
"做一个 JSON 格式化工具，支持语法高亮和错误提示"

✅ 个人博客：
"做一个极简博客，支持 Markdown 文章，标签分类，暗色模式"
```

### SOLO 的边界

| SOLO 适合 | SOLO 不适合 |
|---|---|
| 从零开始的新项目 | 与现有复杂项目集成 |
| 原型验证 | 需要精细业务逻辑 |
| 个人小工具 | 企业级安全要求 |
| 学习练手 | 大规模团队协作项目 |

---

## 模式四：IDE — 智能补全

### 是什么

传统的编码模式 + AI 实时补全（CUE）。

### CUE 智能补全

```
- 代码补全：输入时自动建议，Tab 接受
- 多行修改：AI 识别需要同步修改的多处位置
- 智能导入：自动添加缺失的 import 语句
- 智能重命名：跨文件同步重命名符号
```

### 怎么用

无需任何操作——写代码时自动触发。就像普通的 IDE，但 AI 在你身后。

### 最佳场景

- ✅ 日常编码
- ✅ 熟悉的功能开发
- ✅ 代码整理和重构
- ❌ 需要 AI 理解大范围上下文的场景（用 Chat/Builder）

---

## 四种模式的决策矩阵

```
你现在需要什么？
├── 理解代码 / 问问题 → Chat
├── 在当前项目搭功能 → Builder
├── 从零创建新项目 → SOLO
└── 日常写代码 → IDE（默认）
```

### 实操配合

实际开发中，四种模式是配合使用的：

```
1. SOLO 生成项目骨架
   ↓
2. IDE 模式日常编码（AI 实时补全）
   ↓
3. Builder 快速搭建新功能模块
   ↓
4. Chat 排查 Bug / 获取建议
   ↓
5. 循环
```

---

## 本章小结

1. **Chat** = 问问题，先理解再行动
2. **Builder** = 搭功能，在当前项目快速生成
3. **SOLO** = 建项目，从零生成完整应用
4. **IDE** = 日常编码，AI 在后台默默补全
5. 四种模式是配合使用的，不是互斥的
