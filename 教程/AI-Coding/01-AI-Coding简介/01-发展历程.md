# 1.1 AI Coding 发展历程（2024-2026）

> 理解历史，才能看清趋势。AI Coding 不到三年就走完了从「代码补全」到「多智能体自主协作」的进化路径。本章用时间线帮你建立全局认知。

---

## 概述

AI Coding 的发展可以划分为三个清晰阶段。理解这个演变过程，能帮助你判断：**目前市场上哪些是成熟方案，哪些还在实验期，哪些代表未来方向**。

```
2024: AI 代码补全（Copilot 模式）—— 人主导、AI 辅助
  ↓
2025: Vibe Coding（自然语言→代码）—— 凭感觉编程，Agent 元年
  ↓
2026: Agentic Engineering（多 Agent 自主协同）—— 工程化落地
  ↓
未来: ID Coding（意图编程）—— 人类意图直接转化为软件服务
```

---

## 第一阶段：2024 — 代码补全时代

### 关键事件

| 时间 | 事件 | 为什么重要 |
|---|---|---|
| 2024 年中 | **Claude 3.5 Sonnet** 发布 | 200K 上下文 + 工具使用能力，成为当时最强的编程模型。它证明了「大上下文窗口 + 工具调用」的组合比单纯代码补全强大得多 |
| 2024 年 3 月 | Cognition 推出 **Devin** | 全球首个号称「全自主 AI 软件工程师」的产品，SWE-bench 得分 13.86%，估值冲到 20 亿美元。虽然实际体验远不如宣传，但它开启了「AI 自主编程」的想象空间 |
| 2024 年 | **Cursor** 日活暴涨十倍 | AI 原生 IDE 验证了商业模式，ARR 突破 1 亿美元。证明开发者愿意为更好的 AI 编程体验付费 |
| 2024 年底 | Windsurf (Codeium) 崛起 | ARR 达 1200 万美元，「Vibe Coding」概念开始在开发者社区流行 |

### 阶段特征

这一阶段的核心模式是 **Copilot——人主导、AI 辅助**：

- **交互方式**：开发者在 IDE 里写代码，AI 在旁边补全
- **AI 角色**：被动的代码建议者，不能自主执行任务
- **人类角色**：完全的驾驶者，每一行代码都要自己确认
- **代表产品**：GitHub Copilot、Cursor Tab 补全、早期的 Codeium

> 💡 **理解要点**：2024 年的 AI 还只是个「高级自动补全」。它能帮你写函数体、生成样板代码，但不能理解整个项目，更不会自主做决策。

## 第二阶段：2025 — 智能体（Agent）元年

### 关键事件

| 时间 | 事件 | 为什么重要 |
|---|---|---|
| 2025 年初 | Karpathy 提出 **"Vibe Coding"** | 定义了新范式：用自然语言描述需求，让 AI 生成代码——开发者「凭感觉」编程。这个概念从非共识迅速变成共识 |
| 2025 年 1 月 | 字节跳动发布 **Trae** | 完全免费的 AI 原生 IDE，内置豆包、DeepSeek 等国产模型，迅速占领国内市场 |
| 2025 年初 | Anthropic 发布 **Claude Code** | 终端 CLI 形态的 AI 智能体，不只是聊天——能读文件、写代码、跑命令、管理 Git。6 个月 ARR 突破 10 亿美元 |
| 2025 年 1 月 | **DeepSeek R1** 引爆市场 | 推理能力媲美 OpenAI o1，但成本仅为 1/20，且完全开源。直接冲击了商业模型的定价体系 |
| 2025 年 5 月 | Cursor ARR 达 5 亿美元 | AI IDE 从「尝鲜」变成「主流工具」 |
| 2025 年底 | OpenAI 推出 **Codex**；Google 发布 **Gemini CLI** | 大厂全面入局 Agent 编程，竞争白热化 |
| 2025 年 11 月 | Peter Steinberger 发布 **OpenClaw** | 开源个人 AI Agent，GitHub 24.7 万 Stars。证明「自己跑一个 AI Agent」是可行的 |

### 阶段特征

这一阶段的核心转变：**从补全到 Agent**。

- **交互方式**：从「AI 补全你的代码」变成「你描述需求，AI 自主执行」
- **AI 角色**：从被动的建议者变成主动的任务执行者
- **人类角色**：从写代码的人变成描述需求的人
- **代表产品**：Claude Code、Cursor Composer、Codex、Trae Builder

> 💡 **理解要点**：2025 年最大的变化是 AI 从「帮你写一行」变成了「帮你做一件事」。你不再需要告诉 AI 每步怎么做，只需描述最终效果。

## 第三阶段：2026 — 智能体工程落地年

### 关键事件

| 时间 | 事件 | 为什么重要 |
|---|---|---|
| 2026 年初 | Karpathy 提出 **"Agentic Engineering"** | 多智能体自主完成「规划→编写→测试→审查」全流程。人类从驾驶员变成架构师 |
| 2026 年 2 月 | OpenAI **GPT-5.3-Codex** vs Claude **Opus 4.6** 同夜发布 | 两大模型厂商在编程能力上正面交锋 |
| 2026 年 3 月 | Claude Code 添加 **Computer Use** | AI 可以操作浏览器、桌面应用，能力边界大幅扩展 |
| 2026 年 3 月 | Anthropic ARR 突破 **300 亿美元** | 超越 OpenAI，证明「AI 编程工具」是最大的商业化赛道 |
| 2026 年 4 月 | Cursor 发布 **Cursor 3**，估值 **290 亿美元** | 多智能体协同工作区，AI IDE 进入新阶段 |
| 2026 年 4 月 | Trae 注册用户破 **600 万**，中国市场份额 **41.2%** | 国产工具在国内市场登顶 |
| 2026 年 | Claude Code 获 **46% 开发者最爱** | 在能力认可度上遥遥领先 |

### 阶段特征

这一阶段的核心转变：**从单 Agent 到多 Agent 协同**。

- **交互方式**：多个 AI Agent 分工合作（一个规划、一个编码、一个测试、一个审查），人类只负责定方向和验收
- **AI 角色**：从单个执行者变成协作团队
- **人类角色**：从需求描述者变成架构师和监督者
- **代表产品**：Claude Code 多 Agent、Cursor 3 Background Agents

> 💡 **理解要点**：2026 年 AI 开始像一支小型工程团队。你不需要盯着每一步——Agent 们自己分工、自己检查、自己修正。

## 市场格局（2026 年现状）

| 层级 | 代表 | 核心优势 | 适用人群 |
|---|---|---|---|
| 独立模型厂商 | Anthropic、OpenAI | 模型能力最强，重新定义编程范式 | 追求极致能力的专业开发者 |
| 创业独角兽 | Cursor（$290 亿） | AI 原生 IDE 极致体验 | 从 VS Code 迁移的全能开发者 |
| 平台巨头 | 微软 Copilot、Google | 生态整合 + 分发渠道 | 企业用户、已有微软/Google 技术栈的团队 |
| 国产霸主 | Trae（41.2%） | 免费、合规、中文优化 | 国内开发者首选 |

## 关键趋势（面向未来）

### 1. 工具组合使用成常态

没有人只用一种工具。典型组合：
- **日常编码 80%**：Trae（国内）/ Cursor（海外）
- **架构攻坚 15%**：Claude Code
- **自动化脚本 4%**：Codex
- **个人助手 1%**：OpenClaw / Hermes

### 2. 成本效率成核心考量

Token 消耗爆炸式增长，「无限订阅」模式正在走向崩溃。按用量付费（BYOK——自带密钥）成为高级用户的主流选择。

### 3. 多智能体协作成新范式

一个 Agent 写代码、一个 Agent 测试、一个 Agent 审查——这种分工正在从实验走向工程化。

### 4. 从「会不会用 AI」到「会不会管 AI」

竞争力不再是谁能写出更好的 Prompt，而是谁能设计更好的 Agent 协作流程、制定更精准的验收标准。

---

## 本章小结

1. **2024 年**：AI 只是补全工具，你写代码、它帮忙
2. **2025 年**：AI 成为 Agent，你描述需求、它自主执行
3. **2026 年**：多 Agent 协同，你管方向、它们管执行
4. **未来**：意图编程（ID Coding），你说想要什么、系统直接交付
