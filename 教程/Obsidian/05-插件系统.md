## 六个核心插件详解

### 1. Templater — 动态模板

比 Obsidian 内置模板强在：动态日期、用户输入、JavaScript 执行。

**每日笔记模板**：

```markdown
---
date: <% tp.date.now("YYYY-MM-DD") %>
tags: daily
---

# <% tp.date.now("YYYY年MM月DD日 dddd") %>

## 今日目标
- [ ] 

## 今日记录

## 今日复盘
- 学到了什么：
```

**读书笔记模板**（含弹窗输入）：

```markdown
---
title: <% await tp.system.prompt("书名") %>
author: <% await tp.system.prompt("作者") %>
status: reading
tags: book
date_started: <% tp.date.now("YYYY-MM-DD") %>
---

# 《<% tp.frontmatter.title %>》阅读笔记

## 核心观点
## 关键摘录
## 个人思考
```

### 2. Dataview — 笔记数据库

被称为「Obsidian 中的 SQL」，用查询语句动态生成列表和表格。

**列出所有带某标签的笔记**：
````markdown
```dataview
LIST
FROM #待办
```
````

**按状态查询项目**：
````markdown
```dataview
TABLE status, priority, due
FROM "01-Projects"
WHERE status = "进行中"
SORT priority DESC
```
````

**最近 7 天创建的笔记**：
````markdown
```dataview
TABLE file.cday AS "创建日期", file.tags AS "标签"
WHERE file.cday >= date(today) - dur(7 days)
SORT file.cday DESC
```
````

**典型应用**：
- 项目看板——汇总项目笔记，按状态筛选
- 阅读清单——汇总读书笔记，按评分排序
- 知识库仪表盘——本周新增、进行中项目、待处理收件箱

### 3. QuickAdd — 快速捕获

四种模式：

| 模式 | 用途 | 示例 |
|---|---|---|
| **Template** | 一键按模板创建笔记 | 一键创建会议笔记 |
| **Capture** | 快速追加内容到指定文件 | 闪念捕获→当日日记 |
| **Macro** | 链式执行多个操作 | 创建笔记→应用模板→在日记中加链接 |
| **AI** | 通过 AI 命令处理 | 一键从 URL 生成阅读笔记 |

**闪念捕获配置**：
```
1. 创建 Capture 类型 Choice
2. 目标：当日日记的"灵感捕获"章节
3. 绑定快捷键 Cmd+Shift+I
4. 任何时候按快捷键 → 弹窗输入 → 自动追加到日记（带时间戳）
```

### 4. Calendar — 日历导航

- 右侧边栏显示月历
- 点击日期打开/创建当日日记
- 有日记的日期显示小圆点
- 配合 Periodic Notes 做周记/月记

### 5. Custom Attachment Location — 图片管理

**解决的问题**：Obsidian 默认图片散落各处，改名时链接断裂。

**配置**：

| 配置项 | 推荐值 |
|---|---|
| Attachment folder path | `assets/${noteFileName}` |
| Markdown URL 格式 | `assets/${noteFileName}/${generatedAttachmentFileName}` |
| Generated name | `file-${date:YYYY-MM-DD-HHmmss}` |
| Rename 附件 | 全部勾选 |

**效果**：
```
笔记重命名 → assets/子文件夹自动改名 → 图片链接自动更新
```

### 6. Enhancing Export — 多格式导出

依赖 Pandoc，将笔记导出为 Word/PDF/HTML/ePub/LaTeX。

```bash
# macOS 安装 Pandoc
brew install pandoc
```

右键笔记 → 导出为 → 选择格式即可。

## 插件性能建议

- **20 个左右**的活跃插件是平衡点
- 安装 Lazy Plugin Loader 延迟加载不常用插件
- 定期审视插件列表，禁用/删除不再使用的
- 某个插件明显拖慢速度时，先找更轻量的替代品
