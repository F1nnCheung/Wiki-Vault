# Obsidian 03 — 核心功能详解

> Obsidian 的核心价值不是好看的主题或海量插件——而是双向链接、图谱视图、反向链接这三大机制。它们共同构建了一个网状的知识系统。

---

## 双向链接：知识网络的基石

### 什么是双向链接

传统的超链接是**单向的**——A 链接到 B，但 B 不知道 A 链接了他。Obsidian 的双向链接让每个连接都是双向的。

```
传统笔记：A → B（A 知道 B，B 不知道 A）
Obsidian：A ↔ B（双方都知道这个关系）
```

### 创建链接

```markdown
[[笔记名]]           → 链接到同名笔记
[[笔记名|显示文字]]   → 链接到笔记，显示不同文字
[[笔记名#标题]]      → 链接到笔记的特定章节
[[笔记名#^段落ID]]   → 链接到笔记的特定段落
```

**实操**：在任意笔记中输入 `[[`，Obsidian 会自动弹出笔记列表供选择。

### 链接的最佳实践

```
✅ 在正文中自然嵌入链接：
"关于这个话题，[[PARA方法]] 提供了不同的视角"

✅ 在笔记底部添加「相关笔记」区块：
## 相关笔记
- [[原子化笔记]]
- [[Zettelkasten方法]]

❌ 不要为了链接而链接——每条链接都应有意义
```

## 图谱视图：看见你的思维

### 图谱是什么

图谱视图将所有笔记可视化为节点网络——每个节点是一篇笔记，每条连线代表一个 `[[链接]]`。

```
打开方式：Cmd/Ctrl + G 或左侧边栏"图谱视图"图标
```

### 如何阅读图谱

```
🌑 孤立节点（无连线）→ 这篇笔记需要链接到其他笔记
🌐 密集节点群       → 这是你的知识「热点区域」
🕸️ 桥接节点        → 连接两个不同知识领域的枢纽
📏 长距离连线       → 可能隐藏着意想不到的关联
```

### 图谱的使用时机

- **每周**：看看哪些笔记是孤立的，补充链接
- **每月**：观察知识库的增长模式
- **灵感枯竭时**：漫游图谱，发现之前没注意到的关联

> 💡 图谱是发现工具，不是导航工具。日常导航用搜索和反向链接，图谱用来看全局。

## 反向链接：不被单向性蒙蔽

### 什么是反向链接

当你打开一篇笔记，Obsidian 自动告诉你「哪些笔记链接到了这里」。这是 Zettelkasten 方法的核心——知识因为被引用而产生价值。

```
打开任意笔记 → 右侧边栏 → "反向链接"面板
```

### 链接 vs 反向链接

| | 正向链接（出链） | 反向链接（入链） |
|---|---|---|
| 含义 | 我认为这和什么相关 | 什么认为这和它相关 |
| 可见性 | 在笔记正文中写明的 | 自动聚合的 |
| 价值 | 表达自己的理解 | 发现被动的关联 |

### 典型场景

```
你写了一篇 [[决策疲劳]] 的笔记，链接到了 [[意志力]]。

当你打开 [[意志力]] 时，反向链接面板自动显示：
→ [[决策疲劳]] 引用了这里

这就是「涌现」——你没有显式地在 [[意志力]] 里提 [[决策疲劳]]，
但系统帮你发现了这个关系。
```

## 标签系统：多维度分类

### 标签 vs 链接

| | 标签 `#tag` | 链接 `[[note]]` |
|---|---|---|
| 用途 | 分类、状态标记 | 概念关联 |
| 粒度 | 粗（一个大类） | 细（一个具体概念） |
| 举例 | `#待处理` `#book` `#心理学` | `[[认知偏差]]` `[[PARA方法]]` |

### 标签命名规范

推荐使用**嵌套标签**：

```markdown
#领域/心理学
#领域/计算机科学

#类型/书籍
#类型/文章
#类型/视频

#状态/草稿
#状态/已完成
#状态/待复习

#来源/微信公众号
#来源/RSS
#来源/网页剪藏
```

### 标签面板

开启「标签面板」核心插件后，左侧边栏会显示所有标签及其使用频率。标签越常用，字体越大——快速定位你的知识热点。

## YAML Frontmatter：笔记的身份证

### 什么是 Frontmatter

每篇笔记顶部的 `---` 包裹的 YAML 数据块，用于存储结构化元数据：

```yaml
---
title: 认知偏差列表
tags: [心理学, 认知科学]
created: 2026-05-12
status: 草稿
---
```

### 推荐字段

| 字段 | 用途 | 示例 |
|---|---|---|
| `title` | 笔记标题 | `认知偏差列表` |
| `tags` | 标签列表 | `[心理学, 认知科学]` |
| `created` | 创建日期 | `2026-05-12` |
| `updated` | 最后更新 | `2026-05-12` |
| `status` | 状态标记 | `草稿 / 已完成 / 待复习` |
| `source` | 来源 | `某本书 / 某篇文章` |

### 为什么重要

- **Dataview 插件**可以根据 frontmatter 查询和筛选笔记
- **结构化数据**让 AI 更容易理解和操作你的知识库
- **元数据管理**：笔记多了以后，仅靠标题和标签不够

## 搜索：比链接更快的入口

### 快速切换

```
Cmd/Ctrl + O → 输入关键词 → 实时搜索笔记名
```

这是 Obsidian 中**最高频**的操作。

### 全局搜索

```
Cmd/Ctrl + Shift + F → 搜索所有笔记的正文
```

支持高级搜索语法：

```
path:日记            → 只搜"日记"文件夹
tag:#心理学          → 搜特定标签
line:(TODO)          → 搜特定行内容
file:".md"           → 只搜 Markdown 文件
```

## 核心功能协作实例

假设你刚读完一本关于习惯养成的书：

```
1. 创建 [[原子习惯]] 笔记，写下核心观点（用自己的话）
2. 在正文中链接到已有的笔记：
   → "这和 [[习惯回路]] 的概念是一致的"
   → "与 [[决策疲劳]] 有密切关系"
3. 添加 frontmatter：
   tags: [心理学, 习惯, 书籍]
   status: 草稿
4. 打开图谱视图 → 看到 [[原子习惯]] 连接到了已有的知识群
5. 打开 [[决策疲劳]] → 反向链接面板自动显示 [[原子习惯]] 引用了它
```

## 本章小结

1. **双向链接**：`[[笔记名]]`，让知识形成网络而非孤岛
2. **图谱视图**：可视化你的思维，发现孤立和热点
3. **反向链接**：自动发现「谁引用了我」
4. **标签系统**：嵌套标签 `#领域/子类`，多维度分类
5. **Frontmatter**：YAML 元数据，笔记的身份证
6. **搜索**：`Cmd+O` 快速切换是最常用的操作
