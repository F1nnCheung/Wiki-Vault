# Obsidian 08 — AI 集成方案

> 2026 年最重要的范式转变：整理知识不再只是人的责任。AI 可以帮你摄入、整理、检索、格式化——你只需负责策展和方向。

---

## 为什么 Markdown + AI 是绝配

1. **Markdown 是 LLM 的母语**：`##`、`>`、` ``` ` 等标记本身就是结构分隔符，token 效率比 JSON/XML 高 30-50%
2. **本地文件 = AI 直接读写**：不需要 API、不需要认证
3. **三个十亿级项目殊途同归**：Manus、OpenClaw、Claude Code 都在用 Markdown 文件做 Agent 记忆层

---

## 六种方案总览

| 方案 | 组合 | 成本 | 隐私 | 门槛 | 适合 |
|---|---|---|---|---|---|
| **Claude Code 直连** | Claude Code + Vault | 中 | 高 | 中高 | 追求最强能力 |
| **Copilot + DeepSeek** | Copilot 插件 + DeepSeek API | 极低 | 中 | 低 | 新手入门 |
| **Claudian 插件** | Claudian + Claude Code | 中 | 高 | 中 | 侧边栏无缝体验 |
| **Copilot + Ollama** | Copilot 插件 + Ollama | 免费 | 极高 | 中高 | 隐私敏感 |
| **Gemini CLI** | Gemini CLI + Vault | 低 | 中 | 中 | Google 生态 |
| **LLM Wiki** | Claude Code + raw/wiki 架构 | 中 | 高 | 高 | 知识库自动化 |

---

## 方案一：Claude Code 直连（最强能力）

**适合**：追求最强 AI 能力的深度用户。

### 搭建（30 分钟）

```bash
# 第一步：cd 进 Vault
cd ~/Documents/MyBrain
claude

# 第二步：在 Vault 根目录创建 CLAUDE.md
# 告诉 AI：Vault 结构、frontmatter 模板、行为规则

# 第三步：每个大目录放 index.md（3-5 行概述）

# 第四步：安装 obsidian-skills（处理 [[]] 等特有语法）
git clone https://github.com/kepano/obsidian-skills.git .claude/skills/obsidian-skills
```

### 实战：让 AI 整理散乱笔记

```
你："帮我整理 raw/ 下的散乱笔记。读取所有文件，
   按主题分类，给每条笔记加 Front Matter，
   建立 [[双向链接]]，最后给每个文件夹生成 index.md。"
```

### 进阶方向

1. **raw → wiki → output 三层架构**：AI 把原始素材编译成结构化 wiki
2. **自动 backlinks**：AI 扫描日记中人名/地名/概念，自动加 `[[wikilink]]`
3. **嵌进 Obsidian**：Claudian 插件把对话界面嵌入侧边栏

---

## 方案二：Copilot + DeepSeek（新手最佳）

**适合**：低成本入门，不想折腾配置。

### 为什么 DeepSeek

- **便宜**：比 Claude API 便宜 6-15 倍，充 10 元用很久
- **简单**：Copilot 插件界面化配置
- **够用**：总结、翻译、基础处理完全满足

### 搭建

1. Obsidian 插件中心搜 **Copilot** → 安装
2. deepseek.com 创建 API Key（充值 10 元）
3. Copilot 设置 → Model → 选 deepseek-chat → 填入 API Key 和 Base URL
4. Default Chat Model 选择 deepseek_chat

---

## 方案三：Claudian 插件（无缝体验）

把 Claude Code 对话嵌入 Obsidian 侧边栏，避免 Alt+Tab 切窗口。

**安装**：从 GitHub（github.com/YishenTu/claudian）下载插件文件放入 `.obsidian/plugins/`。

**关键特性**：
- 完整支持 Claude Code Skills（`/` 命令面板）
- 内联编辑：选中文本，AI 直接在原文中修改
- 支持 obsidian-skills 官方 Skill 集

---

## 方案四：Copilot + Ollama（完全本地）

**适合**：对隐私极度敏感，零数据外传。

```bash
# 安装 Ollama
brew install ollama

# 拉取模型
ollama pull llama3.2        # 对话模型
ollama pull nomic-embed-text # 嵌入模型

# 启动
ollama serve
```

Copilot 设置中选择 Ollama 作为提供商。

> 这是 Obsidian 在 AI 时代的杀手锏组合：**本地 Markdown + 本地 LLM = 完全私有的智能知识系统**。Notion AI 无论如何做不到。

---

## 方案五：Gemini CLI（Google 出品）

Google 的命令行 AI 工具，直接操作本地文件系统。

```bash
npm install -g @google/gemini-cli
cd /path/to/your/vault
gemini
# 选择 Login with Google
```

**典型用例**：
- 智能选题：「分析我的历史文章，生成 10 个新选题」
- 批量处理：「把年度总结按二级标题拆分成独立笔记」
- 模仿文风：「学习我的风格，写一篇关于 AI Agent 的脚本」

## 方案六：LLM Wiki 架构（知识库全自动化）

```
Vault/
├── raw/         ← 原始资料（不可变）
├── wiki/        ← AI 生成的结构化知识
└── CLAUDE.md    ← 告诉 AI 如何操作
```

三个核心操作：
- **Ingest**：丢原始资料进 `raw/`，AI 编译成 wiki 页面
- **Query**：问 AI，它先读索引再综合回答
- **Lint**：AI 巡检——发现孤立页面、过时声明、断裂引用

> ⚠️ 一个重要判断：40 万字以内不需要向量数据库。Claude 的上下文窗口够大，纯文本 wiki + index.md 导航，AI 找到相关知识的速度和准确度都很高。

详见 [[../AiCoding/04-拓展使用/01-Obsidian-AI知识库工作流|Obsidian + AI 知识库工作流]]。

## 本章小结

1. Markdown + AI = 绝配，本地文件零摩擦读写
2. 新手从 Copilot + DeepSeek 开始（便宜够用）
3. 深度用户用 Claude Code + obsidian-skills（最强能力）
4. 隐私用户用 Copilot + Ollama（完全本地）
5. 40 万字以内不需要向量数据库
