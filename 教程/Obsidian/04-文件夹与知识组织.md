# Obsidian 04 — 文件夹与知识组织

> 文件放哪里？文件夹怎么建？标签怎么用？这是每个 Obsidian 新手最困惑的问题。本章给出一套可落地的方案。

---

## 核心原则：文件夹不是主角

Obsidian 的核心组织方式不是文件夹，而是**链接**。文件夹只解决「这个文件存在哪里」的问题，链接解决「这个知识关联什么」的问题。

```
文件夹 = 物流仓库（物理位置）
链接   = 知识网络（概念关联）
```

> 💡 不要在文件夹结构上花太多时间。真正的组织靠 `[[链接]]` 和 MOC。

---

## 推荐目录结构（基于 PARA）

```
MyBrain/
├── 00-Inbox/          ← 所有新内容先进入这里
├── 01-Projects/       ← 有明确目标和截止日期的当前工作
├── 02-Areas/          ← 需持续关注的职责范围
├── 03-Resources/      ← 感兴趣的主题、参考资料
├── 04-Archives/       ← 已完成或不再活跃的内容
├── 05-Templates/      ← 笔记模板
├── 06-Attachments/    ← 图片、PDF 等附件
└── 07-Daily/          ← 每日笔记（配合 Calendar 插件）
```

### 各目录详解

**00-Inbox（收件箱）**：
- 一切新内容的第一站
- 不做判断，先存进来
- **每天或每两天清空一次**

**01-Projects（项目）**：
- 有明确目标和截止日期：如「写一篇公众号文章」「搭建个人网站」
- 完成后移动到 04-Archives

**02-Areas（领域）**：
- 没有截止日期但需持续关注：如「健康管理」「技术成长」「财务投资」
- 长期存在，定期回顾

**03-Resources（资源）**：
- 感兴趣的主题：如「机器学习」「设计灵感」「历史」
- 每个资源主题一个子文件夹

**04-Archives（归档）**：
- 已完成的项目、不再关注的领域
- 不删除，只是移到视野之外

**05-Templates（模板）**：
- 配合 Templater 插件使用
- 如每日笔记模板、读书笔记模板、会议笔记模板

**06-Attachments（附件）**：
- 所有图片、PDF 统一存放
- 配合 Custom Attachment Location 插件（见 [[05-插件系统]]）

**07-Daily（日记）**：
- 配合 Calendar 和 Periodic Notes 插件
- 每天一篇，文件名格式如 `2026-05-12.md`

## Zettelkasten 的落地方式

Zettelkasten 强调原子化和链接，不强调文件夹。实际落地时可以这样：

```
不把 Zettelkasten 作为一个独立的工作区。
而是把 PARA 和 Zettelkasten 融合：

Projects / Areas / Resources / Archives  ← PARA（文件夹层）
    └── 每篇笔记内部                       ← Zettelkasten（内容层）
        ├── 原子化（一个想法一篇笔记）
        ├── 用自己的话写
        ├── [[双向链接]] 连接
        └── 标签 + Frontmatter
```

## MOC 导航笔记

### 什么是 MOC

MOC（Map of Content）是一篇**专门用来链接其他笔记**的笔记。它不是文件夹，而是你主动策划的知识地图。

### MOC 示例

```markdown
# 心理学 MOC

## 核心概念
- [[认知偏差]]
- [[决策疲劳]]
- [[习惯回路]]

## 书籍笔记
- [[思考，快与慢]]
- [[原子习惯]]

## 实践方法
- [[如何改善决策质量]]
- [[习惯养成的四个阶段]]

## 待探索
- [[拖延心理学]] ← 还没写，先建一个占位链接
```

### 创建时机

> **不要预先创建 MOC。** 等一个主题的笔记积累到 10-15 篇，自然觉得需要导航了，再创建。

## 标签体系设计

### 推荐维度

```
#领域/xxx      → 知识领域分类
#类型/xxx      → 笔记类型
#状态/xxx      → 完成状态
#来源/xxx      → 信息来源
```

### 标签 vs 文件夹 vs 链接

| | 文件夹 | 标签 | 链接 |
|---|---|---|---|
| 关系 | 一个笔记一个位置 | 一个笔记多个标签 | 一个笔记链接多个笔记 |
| 用途 | 物理存放 | 多维分类 | 概念关联 |
| 粒度 | 最粗 | 中 | 最细 |
| 举例 | 放在 Resources/ | #领域/心理学 #状态/草稿 | [[认知偏差]] [[决策疲劳]] |

> 三者互补，不是替代关系。

## 工作流：从 Inbox 到知识

```
第一步：捕获
  任何新想法、文章、链接 → 直接扔进 00-Inbox
  不需要立即判断价值，先存下来

第二步：处理（每天 15 分钟）
  打开 Inbox → 对每条笔记做决策：
  ├── A 级（立刻有用）→ 移到对应目录，添加链接和标签
  ├── B 级（有参考价值）→ 移到 Resources，加基本标签
  └── C 级（价值低）→ 直接删除

第三步：连接（处理时顺带）
  每条 A 级笔记至少加一个 [[链接]]
  问自己：这条信息和我已知的什么有关？

第四步：回顾（每周 30 分钟）
  打开图谱视图
  ├── 哪些笔记是孤立的？补充链接
  ├── 哪些领域形成了知识集群？考虑创建 MOC
  └── 哪些笔记太久没回顾？重新审视或归档
```

## 常见问题

### Q: 500 篇以上要不要拆 Vault？

**不要急着拆。** 等到两类内容完全不交叉时再考虑（比如个人日记和工作项目完全分开）。Vault 之间的链接是不通的，拆了反而损失了跨领域关联的机会。

### Q: 日记要不要放单独的文件夹？

**建议单独放。** 日记数量增长快，放一起方便管理和搜索。配合 Calendar 插件可视化浏览。

### Q: 应该用多少标签？

没有固定数字。经验法则：**当你在两个标签之间犹豫时，说明粒度太细了。** 合并它们。

## 本章小结

1. 文件夹只是物理存放，真正的组织靠链接
2. PARA 做文件夹骨架，Zettelkasten 做内容方法论
3. MOC 是主动策划的导航笔记，不要预先创建
4. 标签按领域/类型/状态/来源多维度设计
5. 核心工作流：捕获 → 处理 → 连接 → 回顾
