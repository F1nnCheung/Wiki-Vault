# Obsidian 01 — 入门与核心理念

> Obsidian 不只是一个编辑器——它是一套知识管理哲学的技术载体。在使用之前，先理解「为什么」比学会「怎么用」更重要。

---

## Obsidian 是什么

Obsidian 是一个**基于本地 Markdown 文件的知识管理工具**。它的核心特点：

| 特点 | 说明 |
|---|---|
| **本地优先** | 所有笔记保存在你电脑上的 `.md` 纯文本文件中 |
| **双向链接** | 通过 `[[笔记名]]` 语法连接任意笔记，构建知识网络 |
| **图谱视图** | 可视化展示笔记之间的关联关系 |
| **2700+ 插件** | 全球开发者贡献的生态系统 |
| **全平台** | Windows / macOS / Linux / iOS / Android |
| **永久免费** | 个人使用完全免费 |

---

## 核心理念：File over App

Obsidian 的设计哲学可以用一句话概括：

> **你的笔记，你做主。数据永远是你的，工具可以换。**

```
❌ Notion：数据在云端数据库 → 停服了怎么办？导出困难
✅ Obsidian：数据是本地 Markdown 文件 → 任何编辑器都能打开
```

**File over App** 意味着：
- 工具只是 UI 层，底层是纯文本
- 即使 Obsidian 明天消失，你的笔记依然完好
- 可以用 Git 管理版本，可以用任何工具搜索

## 第二大脑（Second Brain）

「第二大脑」不是一个软件，而是一个**系统**——包含工具、方法论和工作流的有机整体。

### 传统定义 vs AI 时代定义

| | 传统第二大脑 | AI 时代第二大脑 |
|---|---|---|
| 核心功能 | 帮助记忆和检索 | AI 理解你、帮助你的上下文来源 |
| 主导者 | 人手动整理 | AI 自动整理，人做策展 |
| 使用方式 | 存入 → 偶尔查阅 | 存入 → AI 加工 → 持续产出 |

> Karpathy 2026 年的核心观点：**停止每次从零推导，开始持续编译知识。** 知识库应该是一个有状态的系统——新知识建立在旧知识之上，不断复利增长。

### 为什么需要第二大脑

我们每天消费大量信息，但大多数人的知识管理方式是「看过即忘」。一个结构化的知识库的价值在于：

1. **不重复思考**：已经想清楚的问题不需要再想一遍
2. **发现关联**：当知识连成网，意想不到的洞察会自然涌现
3. **加速产出**：写作、演讲、决策时，知识库就是你的素材工厂
4. **长期复利**：每天积累一点，一年后是一个庞大的体系

## 三种核心方法论

在你开始使用 Obsidian 之前，先了解这三种知识管理方法论。它们不是互斥的——可以在同一个 Vault 中共存。

### PARA 方法：给知识找到「位置」

由 Tiago Forte 提出，按**可操作性**分四个层级：

| 层级 | 作用 | 示例 |
|---|---|---|
| **P**rojects（项目） | 有明确目标和截止日期 | 「写一篇公众号文章」「搭建个人网站」 |
| **A**reas（领域） | 需持续关注的职责范围 | 「健康管理」「财务投资」「技术成长」 |
| **R**esources（资源） | 感兴趣的主题或参考资料 | 「机器学习」「设计灵感」 |
| **A**rchives（归档） | 已完成或不再活跃的内容 | 去年完成的项目 |

PARA 回答最根本的问题：**这条信息应该放在哪里？**

### Zettelkasten 方法：让知识「连接」起来

德国社会学家 Niklas Luhmann 用这套方法一生出版了 70 多本书。四条核心原则：

1. **原子化**：每条笔记只包含一个想法
2. **用自己的话重写**：永远不要直接复制粘贴
3. **链接优于层级**：通过 `[[双向链接]]` 相连，而非文件夹嵌套
4. **唯一标识**：Obsidian 中文件名天然就是标识符

Zettelkasten 的魔力在于**涌现性**——积累几百条原子笔记并用链接连起来后，会产生意想不到的洞察。

### MOC（Maps of Content）：给知识创建「地图」

MOC 本质上是**导航笔记**——围绕某个主题精心策划的链接列表。

```
MOC 不是文件夹，而是更高层级的笔记。
三五条笔记不需要 MOC，三五十条才需要。
不要预先创建 MOC——让它在需要时自然涌现。
```

---

## 三者如何融合

```
PARA（行动枢纽）        → 文件夹结构，组织项目和领域
Zettelkasten（洞察引擎） → 原子化链接笔记，构建知识网络
MOC（导航层）           → 在两个系统之间架起桥梁
```

三者功能互补、结构互不冲突，可在同一个 Vault 中无缝共存。

---

## 最小可行方案

不需要一步到位。你只需要 5 分钟启动：

1. 安装 Obsidian
2. 创建 Inbox 和 Notes 两个文件夹
3. 每天写一条笔记
4. 每条笔记尽量加一个 `[[链接]]`
5. 装 Calendar 插件方便导航

> **一个最小但持续使用的知识库，比一个完美但三天后就放弃的系统强一万倍。**

---

## 常见误区

| 误区 | 正确做法 |
|---|---|
| 完美体系先行 | 先写笔记，让系统从笔记里长出来 |
| 插件一次性装太多 | 分批进行，每批适应一周再加新的 |
| 过早拆分多 Vault | 500 篇以下别拆，等两类内容完全不交叉时再考虑 |
| 只存不加工 | 知识不是收集出来的，是反复拿出来咀嚼、碰撞、连接出来的 |

## 本章小结

1. Obsidian = 本地 Markdown + 双向链接 + 2700+ 插件
2. File over App：数据永远是你的，工具可以换
3. 第二大脑 = 工具 + 方法论 + 工作流的有机整体
4. 三大方法论：PARA（位置）+ Zettelkasten（连接）+ MOC（导航）
5. 先写起来，系统从实践中长出来
