# 知识库技术 01 — 总览与三代演进

> 知识库技术不是某一个工具或算法，而是一整套**让计算机理解、组织和检索人类知识**的方法论体系。本章帮你建立全局认知地图。

---

## 核心问题：知识库技术解决什么

在开始学技术细节之前，先理解它要解决的**根本矛盾**：

```
人类的知识：分散、隐含、难以复用
计算机的存储：精确、快速、但不懂"意思"

知识库技术 = 在这两者之间架一座桥
```

### 一个典型场景

你过去三年写了 500 篇笔记。某天你想找"关于抗风险的投资策略"，但：
- 有些笔记写的是"杠铃策略"
- 有些写的是"塔勒布的反脆弱"
- 有些写的是"黑天鹅应对"
- 没有任何一篇标题里含"抗风险"

**传统搜索**找不到，因为你搜的词和笔记里的词不一样。

**知识库技术**的目标就是：不管你用什么词问，系统都能理解你的意图，找到相关的内容，甚至告诉你这些内容之间的关联。

---

## 三代演进：从 Ctrl+F 到 AI 推理

知识库技术的发展可以分为三个阶段。理解这个演进过程，你就理解了为什么现在的技术长这样。

### 第一代：关键词搜索（1990s - 2010s）

```
文档 → 分词 → 倒排索引
查询：用户输入关键词 → 索引匹配 → 返回含关键词的文档
```

**代表技术**：Elasticsearch、Lucene、全文索引

**核心能力**：快速找到包含某个词的文档。

**根本局限**：

| 你搜 | 你想找的 | 能找到吗 |
|------|---------|---------|
| "轿车" | 文章里写的"小汽车" | ❌ |
| "怎么理财" | 笔记里写的"资产配置策略" | ❌ |
| "机器学习" | 文章里写的"深度学习"、"神经网络" | ❌ |

关键词搜索不懂**语义**——它只看字面，不理解意思。

### 第二代：语义检索（2020s）

```
文档 → 切块 → Embedding（向量化）→ 向量数据库
查询：用户问题 → Embedding → 语义相似度匹配 → 返回相关内容 → LLM 生成答案
```

这就是 **RAG（检索增强生成）** 的核心。

**代表技术**：向量数据库（Milvus、ChromaDB、Pinecone）、Embedding 模型（OpenAI、BGE、Jina）

**突破**：搜"轿车"能找到"小汽车"，因为它们在向量空间中距离很近。

**新的局限**：

- **不懂关系**："A 的经理的经理是谁？"——答案不在一个段落里，而在关系链中
- **无法推理**："为什么销量下降了？"——需要查多个数据源、综合判断
- **没有积累**：每次查询都从零开始，上次查过的结果不会让下次更聪明

### 第三代：知识理解（2024 - 现在）

```
知识图谱 + Agent + LLM Wiki
```

**三个核心突破**：

| 能力 | 技术 | 解决什么问题 |
|------|------|-------------|
| **理解关系** | 知识图谱 + GraphRAG | 不只找内容，还找实体之间的连接 |
| **自主推理** | Agentic RAG | 不知道该查什么时，AI 自己决定下一步 |
| **知识积累** | LLM Wiki | 每次查询都沉淀为结构化知识，越用越聪明 |

## 技术全景图

```
知识库技术
│
├── 📥 数据摄入层
│   ├── 文档解析（PDF / Markdown / HTML / 代码）
│   ├── 分块策略（语义分块 / 固定大小 / 递归分块）
│   └── 元数据提取（时间 / 作者 / 类型 / 标签）
│
├── 🔢 向量化层
│   ├── Embedding 模型（OpenAI / BGE-M3 / Jina / Ollama）
│   ├── 向量数据库（Milvus / ChromaDB / Qdrant / Pinecone）
│   └── 索引策略（FLAT / IVF / HNSW / DiskANN）
│
├── 🔍 检索层
│   ├── Classic RAG（语义检索）
│   ├── Graph RAG（知识图谱检索）
│   └── Agentic RAG（自主推理检索）
│
├── 🧠 知识组织层
│   ├── 知识图谱（实体 + 关系 + 本体）
│   ├── LLM Wiki（编译器范式：raw → wiki → query）
│   └── 代码知识图谱（AST 解析 + 调用链）
│
└── 🎯 应用层
    ├── 个人知识管理（Obsidian + RAG）
    ├── 企业知识库（内部文档问答）
    ├── 代码理解（AI 编程增强）
    └── 智能家居（自然语言控制）
```

## 本章小结

1. 知识库技术经历了三代：关键词 → 语义检索 → 知识理解
2. RAG 是第二代的核心，但它不懂关系和推理
3. 第三代用知识图谱补关系，用 Agent 补推理，用 LLM Wiki 补积累
4. 本教程覆盖从入门到深入的完整路径，按需选读
