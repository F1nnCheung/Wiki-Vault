# 2.5.3 Codex 适用场景与工具组合

> Codex 的核心优势：边界清楚的活、批量自动化、代码审查 Debug——这些都是它 token 效率 4 倍优势最能发挥的场景。与 Claude Code 不是二选一，而是看活下菜。

---

## 适合 Codex 的场景

### ✅ 1. 从零搭建全栈项目

Codex 可以一条龙完成：iOS App（Swift）+ 落地页（React）+ 数据库（Supabase）+ 部署（Vercel/Netlify）。典型案例：单人创业者几小时内拿到能上 TestFlight 的产品 + 投资人 Deck + 推广视频。

### ✅ 2. 代码审查与 Debug

Codex 逻辑精确、token 省，做 review 比 Claude Code 性价比高。OpenAI 发布了 `codex-plugin-cc`，可在 Claude Code 中一条斜杠命令调 Codex 做审查。

### ✅ 3. 批量迁移/重构

规则明确、重复性高的工作：
- 「把 src/ 下所有 JS 迁移到 TypeScript」
- 「把 class 组件改成函数组件 + Hooks」

### ✅ 4. 自动化与定时任务

- 每周五下午 4 点汇总 GitHub star 发邮件
- 每天下午 5 点看板进度 + 简报发给老板
- 结合 Computer Use 还能操控桌面应用

### ✅ 5. 非编程任务

- AI 生图（GPT-Image-2）
- 浏览器操控（打开网页、填表、签到）
- 生成视频（Remotion Skill）
- 生成 PPT（电子杂志风 Skill）

### ✅ 6. 云端过夜任务

同步 GitHub 后丢 Codex Cloud，下班丢给它，第二天收 PR。适合「边界清楚、不需要盯」的活。

---

## 不适合 Codex 的场景

### ❌ 1. 从零设计复杂架构

Codex 是执行型工程师，不擅长「从零想架构」。这个阶段用 **Claude Code**，交互推理强、上下文吃得深。

### ❌ 2. 需要精细业务逻辑决策

业务规则往往在人脑里，Codex 擅长执行明确指令，不擅长模糊的业务判断。

### ❌ 3. 安全性要求极高的代码

涉及支付、认证等安全关键代码，AI 生成的每一行都需严格人工审查。

## Codex vs Claude Code：看活下菜

| 维度 | OpenAI Codex | Claude Code |
|------|-------------|-------------|
| **范式** | 自主 Agent，沙箱跑完给你看 | 交互 Copilot，边干边商量 |
| **气质** | 执行型工程师 | 协作者 |
| **Token 效率** | 约 4 倍优势，省额度 | 推理深，但烧得快 |
| **SWE-bench Verified** | GPT-5.5 领先 **88.7%** | 87.6% |
| **SWE-bench Pro** | 58.6% | Opus 4.7 领先 **64.3%** |
| **最适合** | 边界清楚的活、review、debug、批量 | 从零想架构、初始功能、探索 |

### 高手分工

```
Claude Code：架构决策 + 初始功能生成（交互推理强、上下文深）
    +
Codex：代码审查 + Debug + 批量自动化（逻辑精确、token 省）
    +
Cursor/Trae：日常实时代码补全（Tab 补全体验）
```

OpenAI 2026 年 3 月发布 `codex-plugin-cc`，在 Claude Code 中直接调 Codex 做审查，两个工具已打通。

## 场景决策矩阵

```
你的任务是什么类型？
├── 从零设计架构 / 探索性开发 → Claude Code
├── 从零搭建全栈项目（执行向） → Codex ⭐
├── 代码审查 / Debug → Codex ⭐
├── 批量迁移 / 重构 → Codex ⭐
├── 自动化 / 定时任务 → Codex ⭐
├── 非编程任务（生图/PPT/视频） → Codex ⭐
├── 云端过夜任务 → Codex Cloud ⭐
├── 日常实时代码补全 → Cursor / Trae
└── 安全关键代码 → 人工 + 严格审查
```

## 本章小结

1. Codex 的优势场景：边界清楚的执行、批量、自动化、审查——token 效率 4 倍优势最能发挥
2. Claude Code 的优势场景：从零架构、探索、深度推理——交互协作感强
3. 两种工具**互补而非替代**，高手两把都有，看活下菜
4. Codex 不是 4% 的配角——在 2026 年 500+ 人调查中 65% 更倾向用它日常编程
5. codex-plugin-cc 打通了 Codex 和 Claude Code，混用已是标准姿势
