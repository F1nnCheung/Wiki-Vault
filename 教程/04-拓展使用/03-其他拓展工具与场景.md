# 4.3 其他拓展工具与场景

> 除了核心工具和框架，还有一批「小而美」的拓展工具，在特定场景下能大幅提升效率。本章盘点最值得关注的几个。

---

## Figma → 代码：设计稿秒变前端

### 是什么

通过 Figma MCP，Claude Code 可以直接读取 Figma 设计稿，生成对应的前端代码。

### 工作流

```
Figma 设计稿 → Figma MCP → AI 读取设计 → 生成代码
```

### 实操

```bash
# 安装 Figma MCP
claude mcp add figma -- npx @anthropic/mcp-figma

# 在 Claude Code 中
你："读取这个 Figma 文件的设计稿，
   生成对应的 React + Tailwind 组件"
```

### 适用场景

- ✅ 从设计稿快速生成 UI 原型
- ✅ 设计系统 → 代码组件库
- ✅ 配合 FigJam 生成可修改的流程图
- ❌ 需要复杂交互逻辑的页面（AI 只能生成静态 UI）

---

## Playwright MCP：不只是写代码，还能验

### 是什么

Playwright MCP 让 Claude Code 能操作浏览器——打开页面、点击按钮、填写表单、截图验证。

### 完整闭环

```
读代码 → 修改 → 启动本地服务 → 打开页面
  → 操作页面 → 根据结果再修 → 输出验证报告
```

### 典型应用

| 场景 | 操作 |
|---|---|
| 表单验证 | 填写表单 → 提交 → 截图 → 检查错误提示 |
| 登录流程 | 输入账号密码 → 点击登录 → 验证跳转 |
| 管理后台 | 点击按钮 → 检查弹窗 → 验证数据更新 |
| 页面跳转 | 点击链接 → 检查路由 → 验证页面内容 |
| 可访问性 | 检查 aria 标签 → Tab 键导航 → 颜色对比度 |

### 安全规则

```markdown
# CLAUDE.md
## Playwright 安全规则
- 只允许 localhost / 127.0.0.1
- 禁止访问生产环境 URL
- 截图保存到 tests/screenshots/
- 不要在页面上输入真实密码
```

## Claude HUD：可视化监控面板

### 是什么

让 Claude Code 的运行状态一目了然——实时查看 Agent 在做什么，而不是盯着终端看日志。

### 价值

- 📊 可视化当前任务进度
- 🔍 实时查看 Agent 的想法和决策
- 📈 Token 消耗图表
- 🎯 任务完成度追踪

适合长时间运行的 Agent 任务——你不需要一直盯着终端。

## Repomix：一键理解整个项目

### 是什么

将整个代码仓库打包为一个结构化的文件，让 AI 一次性读懂项目全貌。

### 为什么重要

默认情况下，Claude Code 只能逐个读取文件。对于大型项目，这导致：
- 上下文碎片化，AI 看到的是「盲人摸象」
- 多次文件读取消耗大量 Token

Repomix 解决这个问题：一个文件 = 整个项目的结构化视图。

### 使用

```bash
npx skills add repomix

# 在 Claude Code 中
你："用 repomix 分析当前项目，给我项目架构概览"
```

### 适用场景

- 接手新项目，快速理解代码结构
- 大规模重构前的全局分析
- 向 AI 解释复杂的项目架构

## 其他值得关注的方向

### Context7：防 AI 幻觉

自动注入最新框架文档到 AI 上下文中，解决「AI 输出过时 API」的问题。

```bash
claude mcp add context7 -- npx @context7/mcp@latest
```

### Desktop Commander：桌面级操作

让 AI 像人一样操作整台电脑——文件管理、进程控制、系统设置。

### Sequential Thinking：结构化推理

让 AI 在复杂问题上分步推理，而不是一步到位。适合架构设计和算法优化。

---

## 工具选择原则

```
你需要解决什么问题？
├── 设计稿转代码 → Figma MCP
├── 浏览器自动化验证 → Playwright MCP
├── 监控 Agent 运行 → Claude HUD
├── 理解大型项目 → Repomix
├── 防 AI 输出过时 API → Context7
├── 桌面级操作 → Desktop Commander
└── 复杂问题推理 → Sequential Thinking
```

> 💡 不要一次性装所有工具。按项目需求选择 2-3 个。每个工具都会占用上下文和增加复杂度。
