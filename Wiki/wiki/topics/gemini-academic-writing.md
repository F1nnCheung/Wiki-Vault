# Gemini 3.5 + 六个 Skill：学术论文初稿写作工作流

> 将论文初稿的六个核心环节（框架→引言→文献综述→方法→结果分析→结论）封装为六个可复用的 Skill，借助 Gemini 3.5 的长文本能力，实现从零到完整初稿的一气呵成。

## 概述

学术论文初稿写作面临的核心挑战：既要合理组织材料、构建清晰严谨的逻辑体系，又要在学术规范与语言表达之间实现协调统一。

六个 Skill 覆盖论文初稿的完整生命周期，每个 Skill 包含明确的目标（Goals）、约束（Constraints）、能力（Skills）和工作流（Workflows）。核心原则：**AI 辅助而非替代——每个 Skill 都强调个人研究思考的融入和学术不端行为的禁止。**

---

## 六个 Skill 概览

| 步骤 | Skill | 核心产出 | 预计时间 |
|------|-------|---------|---------|
| 1 | **定框架** | 完整论文结构（5 章 + 子标题） | 10 分钟 |
| 2 | **写引言** | 研究背景/意义/现状/思路 | — |
| 3 | **文献综述** | 研究脉络梳理 + 不足分析 + 创新点 | 40 分钟 |
| 4 | **研究方法** | 方法选择依据 + 操作流程 + 数据处理 | — |
| 5 | **结果与分析** | 数据呈现 + 原因分析 + 初步结论 | — |
| 6 | **结论与展望** | 核心成果 + 局限性 + 未来方向 | — |

---

## Skill 1：定框架

**目标**：10 分钟内完成整体大纲搭建，明确各章节逻辑关系。

**核心约束**：
- 必须包含 5 个核心章节：引言、文献综述、研究方法、研究结果与分析、结论与展望
- 每章 3-4 个可展开子标题，避免空泛表达
- 体现研究逻辑递进与章节衔接
- 禁止虚构研究内容

**输入**：研究主题、核心观点、学科领域、论文类型、文献储备情况

> 框架是论文的骨架。框架定好了，后续写作才不会跑偏。

---

## Skill 2：写引言

**目标**：以学术化语言阐述研究问题的现实背景与理论价值。

**核心结构**：
```
研究背景 → 研究意义（理论+实践） → 国内外研究现状 → 研究思路 → 论文结构
```

**关键约束**：
- 国内外研究现状需精准概括核心成果，避免泛泛而谈
- 内容需与研究痛点、研究目标及论文框架高度一致
- 使用严谨规范的学术语言，避免口语化

---

## Skill 3：文献综述

**目标**：系统梳理国内外文献，分析研究不足与空白，突出自身创新性。

**核心方法**：
- 只需提供 3-5 篇核心文献标题或摘要
- 梳理方式：按时间脉络或主题分类
- 评述原则：先总结优势，再分析不足，保持客观性

**关键约束**：
- 禁止虚构文献、堆砌内容或模板化表达
- 引用需符合学科规范（如 GB/T 7714）
- AI 仅辅助，需体现个人研究思考与创新方向

---

## Skill 4：研究方法

**目标**：确保研究过程具有科学性、可操作性与可重复性。

**核心结构**：
```
方法选择依据 → 操作流程（分步骤） → 数据收集与处理 → 适用性分析
```

**关键约束**：
- 明确数据来源、收集工具、处理软件及分析标准
- 操作流程需细节完整具体
- 说明方法适用性与可能局限性

---

## Skill 5：结果与分析

**目标**：系统展示研究结果，深入分析数据背后的原因与关联。

**核心结构**：
```
研究结果呈现 → 数据分析与解读 → 与研究问题的逻辑联系 → 初步结论
```

**关键约束**：
- 分析需基于数据与研究方法，避免无依据推测
- 初步结论需与研究主题、研究问题高度一致
- **禁止虚构数据、夸大结论或篡改案例信息**

---

## Skill 6：结论与展望

**目标**：总结核心成果，分析局限性，提出可操作的未来方向。

**核心结构**：
```
核心结论（3-4 个） → 研究不足（具体+原因） → 未来研究方向
```

**关键约束**：
- 结论不新增未研究内容
- 研究不足需具体明确，说明局限性形成原因
- 未来方向需结合局限性提出，避免空泛
- 禁止夸大研究成果、模板化总结或虚假展望

---

## Skill 设计模式

六个 Skill 共享相同的结构模板：

```yaml
version: 1.0
language: 中文
description: Skill 用途描述
Goals:         # 目标：这个 Skill 要达成什么
Constraints:   # 约束：硬性边界条件
Skills:        # 能力：需要哪些专项能力
Workflows:     # 工作流：输入→处理→输出的步骤
```

这种结构化设计让每个 Skill 既可独立使用，又能串联成完整管线。

---

## 核心心法

1. **AI 定位为辅助工具**：每个 Skill 都明确禁止学术不端（虚构数据/文献/结论），强调个人研究思考的融入
2. **约束比提示更重要**：Constraints 定义了硬性边界，比开放性提示更能保证输出质量
3. **管线化思维**：六个 Skill 串成完整链条，前一步的输出自然成为后一步的输入
4. **时效性适配**：2026 年 AIGC 辅助学术写作规范已被纳入约束条件
