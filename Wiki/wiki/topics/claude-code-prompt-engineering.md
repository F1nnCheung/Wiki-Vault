# Claude 提示词工程

> 核心主张：你每个月付 $20 不是用来让 AI「总结一段文字」的。提示词的底层设计逻辑决定了你能挤出多少价值。

## 五大场景 20 个提示词模板

### 深度研究（4个）

| 模板 | 作用 | 核心逻辑 |
|---|---|---|
| **多源信息融合** | 跨文章提炼共识/冲突/盲区 | 综合而非逐篇总结 |
| **魔鬼代言人** | 四维度无情报打击 | 假设漏洞、执行风险、人性现实、致命硬伤 |
| **钢人模型** | 替反对立场构建最强论证 | 知己知彼，避免认知偏见 |
| **思维模型构建器** | 用 3-5 个模型拆解概念 | 附带案例和常见误区 |

### 写作（4个）

| 模板 | 作用 |
|---|---|
| **文风复刻** | 从句长、词汇、段落结构分析后精准模仿 |
| **铁血主编** | 5 维度审校（删减/软弱观点/缺失/结构/最大硬伤）+ 修改版 |
| **一稿五用** | 同一观点针对小白/专家/杠精/记者/高管写 5 个版本 |
| **提纲变文章** | 碎片笔记→完整文章，不漏观点不加新想法 |

### 职场进阶（5个）

薪资谈判沙盘推演、面试模拟器（含结构化反馈）、简历多岗位定制、艰难对话排练、社交媒体简介重塑

### 日常生活（4个）

7 天食谱 + 购物清单、法律条款大白话翻译、个人账单透视分析、个性化旅行规划

### 高效学习（3个）

| 模板 | 核心机制 |
|---|---|
| **费曼学习法导师** | 逐层追问，不打比方不推进，确保真正理解 |
| **30 天闭环自学** | 80% 实践 + 20% 理论 + 里程碑考核 |
| **苏格拉底式对话** | 只提问不给答案，引导自主思考 |

## Claude Code 提示词最佳实践

### Spec-Driven 开发模式

不是需求文档，而是**接口契约**：

1. 写清楚函数签名、数据结构、边界条件、错误处理方式
2. 命令示例：`"根据 @spec/auth.md 的接口定义，实现 src/auth/ 模块"`
3. 运行测试 + 静态检查，不符合的让 Claude 修正
4. Spec 文件和代码一起提交，下次 Claude 有历史参考

### 成本控制

- 用 `/compact` 定期压缩上下文
- 简单任务用便宜模型（文档整理、格式转换）
- 复杂任务用高质量模型（架构设计、算法优化）
- 合理搭配可降低 70% 以上成本

### 核心原则

1. **先写 CLAUDE.md，再开始开发**：用 `/init` 生成后补充技术栈和编码规范
2. **任务分解比单次大任务更可靠**：设计接口→实现逻辑→写测试→代码审查，每步验证
3. **善用 `@` 引用文件**：`@path/to/file` 比让 AI 自己搜索更精准
4. **用 `/review` + `/security-review` 双重把关**：提交前必跑
5. **把 API Key 放环境变量**：不要写死在配置文件里
