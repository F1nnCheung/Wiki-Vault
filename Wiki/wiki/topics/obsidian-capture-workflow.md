# Obsidian 信息收集工作流

信息收集是知识管理的第一步。本文梳理了三种将外部内容快速存入 Obsidian 的方案：Web Clipper（浏览器）、微信同步（Messager/笔记同步助手）、以及收集后的处理闭环。

## 核心原则

> 信息的入口要简单，知识的后处理要深入。

收藏从来不等于保存。真正让一篇文章变成知识库一部分的操作是：**读完 → 理解 → 写感想 → 打标签 → 和已有知识产生链接**。

三种方案都解决同一个问题：把「扔进来」的动作压缩到 5 秒以内，留出精力做真正的消化工作。

## 方案一：Obsidian Web Clipper（浏览器 → Obsidian）

### 是什么

Obsidian 官方出品的 Chrome 浏览器插件，一键将当前网页保存为标准 Markdown 笔记。

### 为什么是「必装」

**安装**：Chrome 应用商店搜索 "Obsidian Web Clipper"，一键安装。

**操作流程**（全程不超过 3 秒）：
1. 在任何网页中点开插件图标
2. 授权打开 Obsidian
3. 文章完整保存进 Vault——标题、正文、图片全保留，格式干净

### 三种内容类型的处理

| 内容类型 | 保存效果 |
|---------|---------|
| **微信公众号文章** | 微信中「在浏览器中打开」→ Web Clipper → 整篇文章完整保存 |
| **YouTube 视频** | **视频直接嵌入 Obsidian 笔记内可播放**，自动带标签和摘要 |
| **普通网页** | 标题 + 正文 + 图片，去除广告和杂乱元素 |

YouTube 视频嵌入是最炸裂的功能——以前收藏几十个教程视频从没认真看完，现在全部进了知识库，变成可被 AI 处理的素材。

### 信息流闭环

```
看到好内容 → 一键 Clip 进 Obsidian → Claude Code 协助消化 → 生成自己的笔记 → 沉淀进知识库
```

以前收藏是终点，现在收藏是起点。信息利用率提升不止十倍。

## 方案二：微信文章同步（微信 → Obsidian）

### 为什么微信文章必须进 Obsidian

微信收藏夹是「信息的坟墓」，不是「知识的土壤」：
- 收藏了就再也不会看
- 搜索体验差
- 无法打标签、链接、引用
- 无法与已有知识产生关联

一篇文章只有进入知识管理系统，被阅读、标注、链接、引用，才真正变成你的知识。

### 选项 A：笔记同步助手

**工作原理**（快递中转站模式）：
1. 在微信里把文章链接发给笔记同步助手（像一个联系人一样发消息）
2. 笔记同步助手自动抓取文章全文、整理格式
3. 文章被送到 Obsidian，存成一条笔记

**技术依赖**：需要 Node.js 运行环境
**Obsidian 端**：需要安装 Local REST API 插件（提供写入通道）

**存进来的笔记格式**：
- 标题自动带日期
- 标签自动加文章来源（`#微信剪藏`）
- 底部留批注区（等你来写第一句感想）

**设计理念**：保存只是起点，不是终点。它把脏活累活干了（抓取、格式化、排版），留一块「我的批注」给你做真正的思考。

### 选项 B：Messager 插件

**对比优势**（vs 手动复制粘贴）：

| 对比维度 | Messager | 手动复制粘贴 |
|---------|----------|------------|
| 同步速度 | 30 秒自动 | 5-10 分钟 |
| 操作步骤 | 3 步 | 8 步以上 |
| 格式保留 | 标题+正文+图片 | 经常丢失格式 |
| 手机端 | 原生小程序 | 不支持 |

**搭建步骤**：
1. 安装 Messager 插件
2. 在 wechatobsidian.com 注册获取 API Key
3. 在 Messager 设置中填入 API Key 和同步目录
4. 微信中打开文章 → 分享到小程序「Obsidian助手」→ 点保存

**费用**：免费版每天 10 条（大部分人够用），年费会员 39.9 元/年

## 方案三：Karpathy Wiki 模式的收集

在 LLM Wiki 架构中，`raw/` 目录天然就是接收外部素材的入口：
- 所有新内容先进入 `raw/inbox/`
- 定期让 AI 处理：读取 → 提取关键信息 → 编译到 `wiki/`
- 人类只需做「判断什么值得存」这个决策

详见 [[Obsidian LLM Wiki 实践|obsidian-llm-wiki-practice]]。

## 收集后的处理闭环：五步知识内化法

入库只是第零步。真正的价值在于后续处理。

### 第一步：快速扫描，判断价值（30 秒）

按 ABC 分级：
- **A 级**：立刻有用，需要深度处理
- **B 级**：有参考价值，先留着
- **C 级**：当时觉得好，现在看一般 → 直接删

这一步帮你过滤掉 80% 的信息噪音。

### 第二步：提取原子笔记（5 分钟）

对 A 级文章，不整篇保留，而是提取核心观点写成独立的原子笔记。每条原子笔记只包含一个核心观点，标题就是观点本身。

### 第三步：建立双向链接（2 分钟）

把新笔记和已有相关笔记用 `[[双链]]` 连接起来。知识不是孤立的，链接越多，价值越大。

### 第四步：打标签分类（1 分钟）

建立多维度标签体系：`#领域/子领域`、`#工具/名称`、`#来源/类型`、`#状态/阶段`

### 第五步：定期回顾，编织知识网（每周 30 分钟）

- 哪些知识点是孤立的？需要补充链接
- 哪些领域形成了知识集群？可以写一篇总结
- 哪些笔记太久没回顾？需要重新审视

## 工作流整合建议

**不要在同一天同时做「收集」和「处理」两件事。**
- 收集时：清空大脑，把链接扔进去就行
- 处理时：放慢速度，认真读，认真想
- 两者需要的状态完全不一样，混在一起两件事都做不好

**推荐节奏**：
- 碎片时间 → 看到好文章 → 一键 Clip/转发
- 每天固定 15 分钟 → 打开收件箱 → 快速处理当天收藏
- 每周固定 30 分钟 → 图谱视图 → 连接、归类、复盘
