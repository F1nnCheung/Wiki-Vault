# ID Coding（意图编程）

## 定义

AI Coding 的终极愿景：人类表达意图 → 直接转化为软件服务。编程语言本身可能消失，取而代之的是意图表达和架构设计能力。

## 演进路径

```
手写代码（2020s之前）
  ↓
AI 代码补全（2024）      Copilot 模式
  ↓
Vibe Coding（2025）      自然语言→代码，凭感觉编程
  ↓
Agentic Engineering（2026）多 Agent 自主协同
  ↓
ID Coding（未来）         意图→服务，编程语言消失
```

## 核心理念

- **意图即代码**：描述「想要什么效果」，而非「如何实现」
- **语言消失**：编程语言从 JavaScript/Python 退化为意图表达的中间层
- **架构为王**：开发者的核心价值从「写代码」迁移到「定架构」
- **AI 执行**：从需求到部署全由 AI 完成

## 对开发者的影响

| 能力 | 重要性变化 |
|---|---|
| 写代码 | ↓ 贬值 |
| 读代码 / 定架构 | ↑ 升值 |
| 系统设计 | ↑ 升值 |
| 需求理解 | ↑ 升值 |
| 代码审查 | → 持续重要 |

## 当前距离

ID Coding 仍是远期愿景。当前最接近的实践：
- **Codex 全流程自动化**：需求→代码→部署
- **Trae SOLO 模式**：一句话生成完整项目
- **Superpowers 闭环**：规范化的 Agent 协作流程
