## 第一章：Obsidian 基本介绍

### 1.1 什么是 Obsidian

Obsidian 是一款基于本地 Markdown 文件的强大知识管理工具。它由两位开发者 Shida Li 和 Erica Xu 于 2020 年创建，最初面向个人知识管理（PKM）和笔记爱好者设计。

与传统的云笔记软件（如 Notion、Evernote、有道云笔记）不同，Obsidian 的核心理念是：

- **你的数据永远属于你**：所有笔记都保存在你本地的电脑上
- **笔记之间可以自由连接**：通过双向链接构建知识网络
- **高度可定制**：通过插件系统将功能扩展到无限可能

Obsidian 的定位不仅仅是"笔记软件"，更是一个**知识操作系统**。它既是笔记编辑器，也是知识连接器，帮助用户将零散的信息点连接成有机的知识网络。

### 1.2 笔记的本质

理解 Obsidian 最重要的一个概念是：**Obsidian 内所有笔记本质上都是本地电脑上的独立 Markdown 文件**。

这意味着：

1. **完全可控**：你的笔记不存储在任何公司的服务器上，你对数据拥有绝对的所有权
2. **永久可用**：即使 Obsidian 开发者停止维护，甚至公司倒闭，你的笔记仍然是普通的 `.md` 文件，可以用任何文本编辑器、Markdown 编辑器打开
3. **高度兼容**：Markdown 是互联网行业最通用的文本格式，几乎所有写作工具、博客平台、静态网站生成器都支持 Markdown
4. **版本可控**：因为笔记是纯文本文件，天然适合用 Git 等版本控制工具进行管理

相比之下，Notion、语雀、印象笔记等云笔记采用的是私有格式（数据库或富文本），一旦平台停止服务，导出和迁移都会非常困难。这种**数据安全感**是任何云端笔记软件都给不了的。

### 1.3 与同类软件相比的核心优势

#### 优势一：数据安全

- 笔记完全本地存储，无需联网即可使用
- 不依赖任何第三方云端服务
- 可自主选择备份方案（Git、网盘、硬盘等）
- 支持端到端加密（通过插件）

#### 优势二：界面丝滑流畅

- 基于 Electron 开发，前端开发者会感到熟悉
- 启动速度快，切换笔记无白屏、无卡顿
- 编辑响应迅速，不会像某些云笔记那样在输入时出现延迟
- 这种流畅感对于保持专注工作时的心流至关重要

#### 优势三：与 AI 工具绝配

- AI Agent（如 Cloud Code、Gemini CLI、Claude Code 等）天生擅长处理本地文件
- Markdown 格式是 AI 最擅长理解和生成的文本格式
- 可以直接用 AI 操作笔记文件，实现智能选题、文风模仿、批量处理等高级玩法

#### 优势四：强大的插件生态

- 目前已有 2000+ 社区插件，涵盖同步、导出、可视化、项目管理等方方面面
- 插件由全球开发者共同维护，功能迭代迅速
- 官方提供完整的插件开发 API，有能力的用户可以开发自己的专属插件

#### 优势五：双向链接与知识图谱

- 这是 Obsidian 区别于普通 Markdown 编辑器的核心特性
- 通过 `[[笔记名]]` 在笔记之间建立双向链接
- 关系图谱（Graph View）将笔记间的连接可视化，帮助发现知识间的隐藏关联
- 这种连接方式更接近人类大脑的思维模式，而非传统的文件夹层级

## 第二章：Obsidian 的安装与界面认识

### 2.1 下载与安装

Obsidian 支持所有主流操作系统：

- **Windows**：官网下载 `.exe` 安装包，双击安装
- **macOS**：官网下载 `.dmg` 文件，拖拽到 Applications 文件夹
- **Linux**：提供 `.AppImage`、`.deb`、`.tar.gz` 等多种格式
- **iOS / Android**：在 App Store 或 Google Play 搜索 "Obsidian" 下载

官网地址：https://obsidian.md

安装后首次启动，你会看到创建新库（Create new vault）或打开已有库（Open folder as vault）的选项。

### 2.2 Vault（库）的概念

Vault 是 Obsidian 中最重要的概念之一。

**Vault 本质上就是一个普通的文件夹**。当你创建一个 Vault 时，Obsidian 会在你指定的位置建立一个文件夹，里面包含：

- 你所有的 `.md` 笔记文件
- 一个 `.obsidian` 隐藏文件夹，里面存放：
  - `app.json`：应用设置
  - `appearance.json`：外观设置
  - `core-plugins.json`：核心插件开关
  - `workspace.json`：当前工作区布局
  - `plugins/`：社区插件文件
  - `themes/`：主题文件
  - `snippets/`：CSS 代码片段

你可以创建多个 Vault，用于不同的场景：

- 一个 Vault 放工作笔记
- 一个 Vault 放学习笔记
- 一个 Vault 放个人日记

Vault 之间完全隔离，切换 Vault 就像切换不同的工作空间。

### 2.3 界面布局介绍

Obsidian 的界面分为几个主要区域：

#### 左侧边栏

- **文件列表（File Explorer）**：以文件夹树的形式展示所有笔记
- **搜索（Search）**：全局搜索笔记标题和内容
- **书签（Bookmarks）**：收藏重要的笔记、文件夹或搜索条件
- **反向链接面板（Backlinks）**：查看链接到当前笔记的所有笔记

点击左侧边栏顶部的图标可以切换不同的面板。

#### 中间编辑区

这是 Obsidian 的核心区域，用于：

- **编辑笔记**：支持所见即所得的实时预览模式（Live Preview）
- **预览笔记**：纯阅读模式
- **分屏编辑**：可以同时打开多篇笔记，左右对照

Obsidian 提供三种编辑模式：

1. **源模式（Source mode）**：显示原始 Markdown 语法
2. **实时预览（Live Preview）**：编辑时实时渲染 Markdown，这是推荐的模式
3. **阅读模式（Reading mode）**：纯阅读，不可编辑

#### 右侧面板

- **大纲（Outline）**：显示当前笔记的标题层级结构，可点击跳转
- **标签面板（Tag pane）**：展示所有标签及其使用情况
- **笔记属性（Properties）**：显示和编辑当前笔记的 YAML 元数据
- **关系图谱局部视图（Local Graph）**：显示与当前笔记相关联的笔记网络

#### 顶部工具栏

- 打开命令面板
- 切换当前 Vault
- 打开关系图谱
- 打开设置
- 新建笔记

#### 状态栏

位于窗口底部，显示：

- 当前笔记的字数、字符数
- 当前文档的阅读时间估算
- 当前光标所在行号
- 模式切换按钮（编辑/预览）

### 2.4 常用快捷操作

掌握快捷键可以大幅提升使用效率：

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/Cmd + P` | 打开**命令面板**，可搜索执行几乎所有操作 |
| `Ctrl/Cmd + O` | **快速切换**，打开笔记搜索框 |
| `Ctrl/Cmd + N` | **新建笔记** |
| `Ctrl/Cmd + E` | 切换编辑/预览模式 |
| `Ctrl/Cmd + Shift + F` | **全局搜索**所有笔记内容 |
| `Ctrl/Cmd + 点击链接` | 在新面板中打开链接的笔记 |
| `Ctrl/Cmd + W` | 关闭当前面板 |
| `Ctrl/Cmd + ,` | 打开设置 |

**命令面板（Ctrl/Cmd + P）** 是最强大的快捷键，你不需要记住所有快捷键，只需要记住这一个，然后在命令面板中搜索你要执行的操作即可。

## 第三章：Obsidian 基本常用设置

### 3.1 外观与主题设置

#### 切换浅色/深色模式

点击左下角的状态栏按钮，或进入 **设置 → 外观 → 基础配色**，选择：

- **Adapt to system**：跟随系统设置
- **Light**：浅色模式
- **Dark**：深色模式

#### 安装社区主题

Obsidian 内置了少量主题，但社区提供了数百款精美主题。

1. 进入 **设置 → 外观 → 主题 → 管理**
2. 点击 **浏览**，打开社区主题市场
3. 浏览并点击 **使用** 安装喜欢的主题

热门主题推荐：

- **Minimal**：现代简洁，高度可定制，适合追求极简的用户
- **Things**：macOS 风格，优雅精致
- **AnuPpuccin**：色彩丰富，自定义选项极多
- **Border**：卡片式设计，视觉层次分明
- **Catppuccin**：柔和的配色方案，护眼舒适

#### 自定义 CSS 片段（进阶）

如果你对 CSS 有了解，可以通过添加 CSS 片段来微调界面：

1. 进入 **设置 → 外观 → CSS 代码片段**
2. 点击文件夹图标打开 `.obsidian/snippets/` 目录
3. 创建 `.css` 文件，写入自定义样式
4. 回到 Obsidian 启用该片段

### 3.2 编辑器设置

进入 **设置 → 编辑器**，常用的选项包括：

- **默认编辑模式**：建议选择 "实时预览"
- **严格换行**：开启后，按一次回车不会换行，需要按两次
- **自动配对括号/引号**：输入左括号自动补全右括号
- **智能列表缩进**：在列表中按 Tab 自动缩进
- **行宽限制**：设置每行最大字符数，超过后自动换行（建议 80-120）
- **显示行号**：在编辑区左侧显示行号

### 3.3 文件与链接设置

进入 **设置 → 文件与链接**，这是影响笔记格式的关键设置：

#### 关闭 Wiki 链接

**强烈建议关闭「使用 Wiki 链接」**，改用标准 Markdown 链接格式。

原因：
- Wiki 链接语法 `[[笔记名]]` 是 Obsidian 特有的，其他编辑器无法识别
- 标准 Markdown 链接 `![显示文字](路径)` 是通用格式，兼容 GitHub、VS Code 等所有工具
- 如果你计划用 Git 同步或分享笔记，标准格式是必须的

设置方法：
- 关闭「使用 Wiki 链接」开关
- 内部链接类型选择「基于当前笔记的相对路径」

#### 其他重要设置

- **新建笔记的存放位置**：设置默认在哪个文件夹下创建新笔记
- **附件默认存放路径**：设置粘贴图片等附件时的默认保存位置
- **使用 Markdown 格式**：确保粘贴内容时转换为 Markdown

### 3.4 核心插件启用

Obsidian 自带了几十个核心插件，部分默认开启，部分需要手动启用。

进入 **设置 → 核心插件**，推荐启用以下插件：

| 插件名 | 功能 | 建议 |
|--------|------|------|
| **日记** | 快速创建和管理每日笔记 | 必开 |
| **模板** | 基于模板创建标准化笔记 | 必开 |
| **快速切换** | `Ctrl/Cmd + O` 快速打开笔记 | 必开 |
| **星标** | 收藏常用笔记 | 推荐 |
| **工作区** | 保存和恢复不同的面板布局 | 推荐 |
| **命令面板** | `Ctrl/Cmd + P` 快捷操作 | 默认开启 |
| **书签** | 标记重要笔记和搜索条件 | 推荐 |
| **发布** | 将笔记发布为公开网页 | 按需 |
| **反向链接** | 查看链接到当前笔记的笔记 | 默认开启 |
| **大纲** | 显示当前笔记的标题结构 | 默认开启 |
| **标签面板** | 管理所有标签 | 推荐 |
| **幻灯片** | 将笔记转为幻灯片演示 | 按需 |
| **录音机** | 在笔记中录制音频 | 按需 |
| **版本历史** | 查看笔记的修改历史 | 推荐 |

启用方法：找到对应插件，将开关打开即可。

## 第四章：笔记的创建与管理

### 4.1 创建笔记的多种方式

Obsidian 提供了多种创建笔记的方式：

1. **点击新建按钮**：左下角的新建笔记按钮
2. **快捷键**：`Ctrl/Cmd + N`
3. **命令面板**：`Ctrl/Cmd + P`，输入 "新建笔记"
4. **文件列表右键**：在文件夹上右键 → 新建笔记
5. **双向链接自动创建**：输入 `[[一个不存在的笔记名]]`，点击即可自动创建

### 4.2 笔记组织方式

Obsidian 提供了三种互补的笔记组织方式，你可以根据自己的习惯组合使用：

#### 文件夹层级

最传统的方式，通过文件夹分类笔记。

优点：
- 直观易懂，符合大多数人的习惯
- 适合层级分明的分类体系

缺点：
- 一个笔记只能放在一个文件夹里
- 分类边界模糊时难以抉择
- 文件夹层级过深会导致查找困难

建议：
- 文件夹不要建太多层，建议不超过 3 层
- 用宽泛的分类代替细碎的分类

#### 标签系统

用 `#标签名` 给笔记打标签，一个笔记可以有多个标签。

```markdown
这是一篇关于 Obsidian 的笔记 #笔记工具 #效率 #知识管理
```

Obsidian 支持**多层级标签**：

```markdown
#编程/JavaScript
#编程/Python
#编程/Go
```

这样 `#编程` 可以搜索到所有编程相关的笔记。

标签 vs 文件夹：
- 标签更灵活，一个笔记可以有多个标签
- 文件夹更稳定，适合存放固定类别的内容
- 建议组合使用：用文件夹做粗略分类，用标签做交叉标记

#### 双向链接

Obsidian 最强大的组织方式。在笔记中输入 `[[笔记名]]` 即可创建一个指向另一篇笔记的链接。

```markdown
我正在学习 [[Obsidian]]，它是一款强大的 [[笔记工具]]。
```

特点：
- 点击链接可以直接跳转到目标笔记
- 目标笔记会自动记录"有哪些笔记链接到了我"（反向链接）
- 即使目标笔记不存在，链接也能创建，点击后会自动新建

#### 嵌入笔记

用 `![[笔记名]]` 可以将另一篇笔记的完整内容嵌入到当前笔记中：

```markdown
下面是某篇笔记的内容：

![[某篇笔记]]
```

这在需要复用内容时非常有用，比如在一个项目总览中嵌入各个子任务的笔记。

**三种笔记组织方式对比**：

```mermaid
graph LR
    subgraph 文件夹层级
        F1[工作] --> F2[项目A]
        F1 --> F3[项目B]
        F2 --> F4[笔记1]
    end

    subgraph 标签系统
        T1[笔记2] -.->|#编程| T2[编程]
        T1 -.->|#JavaScript| T3[JavaScript]
        T4[笔记3] -.->|#编程| T2
    end

    subgraph 双向链接
        N1[笔记A] -->|[[笔记B]]| N2[笔记B]
        N2 -->|[[笔记C]]| N3[笔记C]
        N3 -->|[[笔记A]]| N1
    end
```

| 特性 | 文件夹 | 标签 | 双向链接 |
|------|--------|------|----------|
| 一个笔记的数量限制 | 只能在一个文件夹 | 可以有多个标签 | 可以链接到任意笔记 |
| 适合场景 | 固定分类 | 交叉标记 | 知识关联 |
| 灵活性 | 低 | 中 | 高 |

> **建议**：组合使用三种方式。用文件夹做粗略分类，用标签做主题标记，用双向链接建立知识网络。

### 4.3 YAML 前置属性（Properties）

YAML 前置属性是写在笔记顶部的元数据，用于描述笔记的额外信息。

Obsidian 提供了可视化的属性编辑器，无需手写 YAML。

添加属性的方法：
1. 在笔记顶部点击 "添加属性"
2. 输入属性名和值

常见属性示例：

```yaml
---
title: Obsidian 使用教程
date: 2026-05-06
tags: [笔记工具, 效率, 教程]
category: 知识管理
author: 你的名字
status: 进行中
---
```

属性的用途：
- 为笔记添加结构化元数据
- 配合 Dataview 插件进行筛选和查询
- 配合 Templater 插件自动填充模板字段

### 4.4 搜索与筛选

Obsidian 的搜索功能非常强大。

#### 基本搜索

按 `Ctrl/Cmd + Shift + F` 打开全局搜索面板，输入关键词即可。

#### 搜索语法

| 语法 | 含义 | 示例 |
|------|------|------|
| `"关键词"` | 精确匹配短语 | `"Obsidian 教程"` |
| `-关键词` | 排除包含某词的笔记 | `-归档` |
| `path:文件夹` | 在指定路径下搜索 | `path:编程/JavaScript` |
| `tag:#标签` | 搜索带某标签的笔记 | `tag:#效率` |
| `file:文件名` | 搜索文件名为某值的笔记 | `file:日记` |
| `line:内容` | 搜索行内容 | `line:待办` |
| `section:标题` | 在特定标题下搜索 | `section:参考资料` |

#### 保存搜索

1. 在搜索面板中输入搜索条件
2. 点击右侧的 "复制搜索结果" 或 "创建搜索书签"
3. 书签会出现在左侧边栏的书签面板中，以后一键访问

## 第五章：Markdown 基础语法

Markdown 是一种轻量级标记语言，用简单的符号就能实现丰富的排版效果。Obsidian 对 Markdown 的支持非常完善。

### 5.1 标题语法

用 `#` 表示标题，`#` 的数量代表层级：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**建议**：一篇笔记中只用一个一级标题（即笔记标题），内容用二到四级标题组织。

### 5.2 文本样式

```markdown
**加粗文字**
*斜体文字*
***加粗且斜体***
~~删除线文字~~
==高亮文字==
`行内代码`
```

效果：

- **加粗文字**
- *斜体文字*
- ~~删除线文字~~
- ==高亮文字==
- `行内代码`

> 注意：高亮语法 `==文字==` 不是所有 Markdown 编辑器都支持，但在 Obsidian 中可用。

### 5.3 代码块

行内代码用一个反引号包裹：

```markdown
这是 `console.log()` 的用法
```

多行代码块用三个反引号包裹，并可在第一行指定语言以获得语法高亮：

````markdown
```javascript
function hello() {
    console.log("Hello, Obsidian!");
}
```
````

支持的语言包括：javascript、python、java、bash、json、yaml、css、html 等上百种。

### 5.4 引用与列表

#### 引用

```markdown
> 这是一段引用文字
> 可以有多行
```

效果：

> 这是一段引用文字
> 可以有多行

#### 无序列表

```markdown
- 项目一
- 项目二
  - 子项目
  - 子项目
- 项目三
```

效果：

- 项目一
- 项目二
  - 子项目
  - 子项目
- 项目三

#### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

#### 任务列表

```markdown
- [ ] 待办事项一
- [x] 已完成事项
- [ ] 待办事项二
```

效果：

- [ ] 待办事项一
- [x] 已完成事项
- [ ] 待办事项二

在 Obsidian 中，你可以直接点击复选框来标记完成状态。

### 5.5 插入元素

#### 表格

```markdown
| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25   | 北京 |
| 李四 | 30   | 上海 |
| 王五 | 28   | 深圳 |
```

效果：

| 姓名 | 年龄 | 城市 |
|------|------|------|
| 张三 | 25   | 北京 |
| 李四 | 30   | 上海 |
| 王五 | 28   | 深圳 |

在 Obsidian 中，你也可以右键 → 插入 → 表格，通过图形界面创建。

#### 分隔线

```markdown
---
```

#### 数学公式

Obsidian 支持 LaTeX 数学公式：

行内公式：

```markdown
这是勾股定理：$a^2 + b^2 = c^2$
```

块级公式：

```markdown
$$
E = mc^2
$$
```

效果：

$$
E = mc^2
$$

#### 脚注

```markdown
这是一个有脚注的句子[^1]。

[^1]: 这是脚注的内容。
```

### 5.6 链接与图片

#### 外部链接

```markdown
[显示文字](https://example.com)
```

#### 内部链接（Wiki 链接）

```markdown
[[另一篇笔记]]
[[另一篇笔记|自定义显示文字]]
```

#### 图片

```markdown
![图片描述](图片路径.png)
```

在 Obsidian 中，你可以直接粘贴图片，会自动保存到设置的附件文件夹中。
