# Obsidian 使用教程大纲

---

## 第一部分：Obsidian 基础

### 第一章：Obsidian 基本介绍

#### 1.1 什么是 Obsidian
- 一款基于本地 Markdown 文件的强大知识管理工具
- 既是笔记编辑器，也是知识连接器

#### 1.2 笔记的本质
- 所有笔记都是本地电脑上的独立 Markdown 文件
- 即使 Obsidian 开发者跑路，笔记也不会丢失，换任何 Markdown 编辑器即可继续使用
- 数据独立、可迁移，这种安全感是任何云端笔记软件都给不了的

#### 1.3 与同类软件相比的核心优势
- **数据安全**：本地存储 + 可自主备份，不依赖第三方云端
- **界面丝滑流畅**：切换笔记无白屏、无卡顿，保持专注工作时的心流
- **与 AI 工具绝配**：AI Agent（如 Cloud Code、Gemini CLI 等）天生擅长处理本地 Markdown 文件
- **强大的插件生态**：目前已有 2000+ 社区插件，可无限扩展功能
- **双向链接与知识图谱**：颠覆传统文件夹式笔记管理，构建知识网络

### 第二章：Obsidian 的安装与界面认识

#### 2.1 下载与安装
- 官网下载对应操作系统版本（Windows / macOS / Linux / iOS / Android）
- 安装后首次启动：创建新库（Vault）或打开已有库

#### 2.2 Vault（库）的概念
- Vault 就是一个普通文件夹，里面存放所有 .md 笔记文件
- `.obsidian` 配置文件夹：存放设置、主题、插件等配置文件
- 可创建多个 Vault，用于不同场景（如工作、学习、生活）

#### 2.3 界面布局介绍
- **左侧边栏**：文件列表、搜索、书签、反向链接面板
- **中间编辑区**：撰写和编辑笔记的主要区域（支持编辑/预览双栏模式）
- **右侧面板**：大纲、标签面板、笔记属性、关系图谱等
- **顶部工具栏**：打开命令面板、切换资料库、关系图谱、设置等快捷按钮
- **状态栏**：显示当前字数、阅读模式切换、当前文档信息等

#### 2.4 常用快捷操作
- **命令面板**：`Ctrl/Cmd + P`，搜索并执行几乎所有操作
- **快速切换**：`Ctrl/Cmd + O`，快速打开笔记
- **搜索**：`Ctrl/Cmd + Shift + F`，全局搜索笔记内容
- **新建笔记**：`Ctrl/Cmd + N`
- **切换编辑/预览模式**：`Ctrl/Cmd + E`

### 第三章：Obsidian 基本常用设置

#### 3.1 外观与主题设置
- 内置浅色/深色主题切换
- 社区主题市场：Minimal、Things、AnuPpuccin 等热门主题
- 自定义 CSS 片段（进阶）

#### 3.2 编辑器设置
- 行宽限制、缩进、字体大小
- 严格换行、智能列表、自动配对括号等

#### 3.3 文件与链接设置
- 关闭「使用 Wiki 链接」，确保标准 Markdown 链接语法 `![ ]()`
- 内部链接类型选「基于当前笔记的相对路径」
- 新建笔记的存放位置、附件默认存放路径

#### 3.4 核心插件启用
- **日记**：快速创建每日笔记
- **模板**：基于模板快速创建标准化笔记
- **快速切换**：`Ctrl/Cmd + O` 快速打开笔记
- **星标**：收藏常用笔记
- **工作区**：保存和恢复不同的面板布局
- **命令面板**：`Ctrl/Cmd + P` 快捷操作入口
- **书签**：标记重要笔记和搜索条件
- **发布**：将笔记发布为公开网页（Obsidian Publish）
- **反向链接**：查看链接到当前笔记的所有笔记

### 第四章：笔记的创建与管理

#### 4.1 创建笔记的多种方式
- 点击新建按钮、`Ctrl/Cmd + N`、命令面板
- 在文件列表中右键新建
- 通过双向链接 `[[新笔记名]]` 自动创建不存在的笔记

#### 4.2 笔记组织方式
- **文件夹层级**：传统的目录树结构
- **标签系统**：用 `#标签名` 给笔记打标签，支持多层级标签（如 `#编程/Java`）
- **双向链接**：用 `[[笔记名]]` 在笔记之间建立关联
- **嵌入笔记**：用 `![[笔记名]]` 将另一篇笔记的内容嵌入当前笔记

#### 4.3 YAML 前置属性（Properties）
- 在笔记顶部添加元数据，如标题、日期、标签、分类等
- 支持搜索和筛选，为后续 Dataview 等插件提供数据基础
- 可视化属性编辑器，无需手写 YAML

#### 4.4 搜索与筛选
- 全局搜索：`Ctrl/Cmd + Shift + F`
- 搜索语法：`"关键词"`、`-排除词`、`path:路径`、`tag:#标签`、`file:文件名`
- 保存常用搜索条件到书签

### 第五章：Markdown 基础语法

#### 5.1 标题语法
- `# 一级标题`、`## 二级标题`、`### 三级标题`，以此类推

#### 5.2 文本样式
- **加粗**：`**加粗**` 或 `****加粗****`
- **斜体**：`*斜体*` 或 `_斜体_`
- **删除线**：`~~删除线~~` 或 `~~~~删除~~~~`
- **高亮**：`==高亮内容==` 或 `====高亮====`

#### 5.3 代码块
- 行内代码：`` `代码` ``
- 代码块：用 ``` 包裹，第一行指定语言，如 ```java、```python

#### 5.4 引用与列表
- **引用**：`> 引用内容`
- **无序列表**：`- 项目` 或 `* 项目`
- **有序列表**：`1. 项目`
- **任务列表**：`- [ ] 待办事项`、`- [x] 已完成事项`

#### 5.5 插入元素
- **表格**：右键插入或使用 `|` 手动绘制
- **分隔线**：`---`
- **数学公式**：行内 `$公式$` 或块级 `$$公式$$`，支持 LaTeX 语法
- **脚注**：`[^1]` 和 `[^1]: 脚注内容`

#### 5.6 链接与图片
- **外部链接**：`[显示文字](URL)`
- **内部链接（Wiki 链接）**：`[[笔记名]]` 或 `[[笔记名|显示文字]]`
- **图片**：`![描述](图片路径)`

## 第二部分：Obsidian 进阶用法

### 第六章：核心进阶功能（Obsidian 自带功能）

#### 6.1 双向链接与知识图谱
- **双向链接**：输入 `[[笔记名]]` 创建链接，自动建立笔记间的双向关联
- **关系图谱（Graph View）**：可视化展示所有笔记的链接关系
  - 全局图谱：查看整个知识库的网络结构
  - 局部图谱：查看与当前笔记相关的笔记网络
- **反向链接面板**：在笔记底部自动显示链接到当前笔记的所有笔记
- **未链接提及**：发现正文中提到但未建立链接的笔记名

#### 6.2 Canvas 画布功能
- **功能**：无限画布，可在上面放置笔记卡片、图片、网页嵌套，用线条连接
- **场景**：头脑风暴、项目规划、知识梳理、流程图绘制
- **操作**：新建 Canvas、添加卡片、拖拽排版、建立连接

#### 6.3 工作区（Workspace）管理
- **功能**：保存当前面板布局（打开的文件、侧边栏、分屏等）
- **场景**：为不同任务保存专属布局（写作模式、阅读模式、研究模式）
- **配合 Git**：将 workspace.json 加入 `.gitignore`，避免多端冲突

### 第七章：笔记组织方法论（通用方法）

#### 7.1 PARA 方法
- **Projects（项目）**：当前正在进行的任务，有明确截止日期
- **Areas（领域）**：需要长期维护的责任领域（如健康、财务、技能）
- **Resources（资源）**：感兴趣的主题和参考材料
- **Archives（归档）**：已完成或暂停的项目和领域

#### 7.2 MOC（Map of Content）内容地图
- 用一篇索引笔记作为某个主题的导航入口
- 汇总该主题下的所有相关笔记和链接
- 解决文件夹层级僵化的问题

#### 7.3 原子笔记（Atomic Notes）
- 每条笔记只记录一个核心概念或想法
- 通过链接将原子笔记组合成复杂知识
- 提高笔记的复用性和可组合性

### 第八章：云同步插件 —— Git（第三方，电脑端 + 手机端完整教程）

#### 8.1 为什么需要云同步
- Obsidian 本身不提供云同步，需自行解决多端同步问题
- 方案选择：GitHub（完全免费，比任何网盘都安全稳定）

#### 8.2 GitHub 仓库准备
- 注册 GitHub 账号
- 创建私有仓库（如 personal_vault），避免笔记公开到互联网

#### 8.3 仓库克隆到本地（电脑端）
- 使用 GitHub Desktop 客户端（新手友好）或命令行
- 将远程仓库克隆为本地文件夹
- 在 Obsidian 中选择 "Open local vault" 打开该文件夹

#### 8.4 Git 基础配置
- **.obsidian 文件夹**：存放 Obsidian 配置文件
- 创建 `.gitignore` 文件，排除 `.obsidian/workspace.json` 和 `.obsidian/workspace-mobile.json`
- 避免工作区状态文件频繁变动造成 Git 冲突

#### 8.5 Git 插件安装与配置（电脑端）
- 在社区插件市场搜索安装 **Git** 插件（Vincent, Denis Olekhov）
- 核心配置项：
  - **Auto commit-and-sync after stopping file edits**：停止编辑后自动同步
  - **Auto commit-and-sync interval**：设置间隔（如 1 分钟）
  - **Pull on startup**：启动 Obsidian 时自动从 GitHub pull 更新
  - **Push on commit-and-sync / Pull on commit-and-sync**：均开启
- 实现全自动同步，省去手动提交步骤

#### 8.6 双向同步验证（电脑端）
- 本地修改自动推送到 GitHub，新文件自动出现在网页端
- GitHub 网页端修改自动 pull 回本地，Obsidian 右上角弹窗提示
- 确保多端笔记双向一致

#### 8.7 手机端初始文件同步
- 用数据线连接手机与电脑，选择传输文件模式
- 将电脑 Vault 文件夹复制到手机 Documents 目录

#### 8.8 手机端 Obsidian 设置
- 下载 iOS 或 Android 版 Obsidian App
- 打开 App → Open folder as vault → 选择 Documents/Vault 文件夹
- 信任作者并启用插件，避免受限模式

#### 8.9 手机端 Git 插件配置
- 进入 Git 设置，填写：
  - **Author name**：GitHub 用户名
  - **Email**：注册邮箱
  - **Personal access token**：在 GitHub 生成
- 电脑端 GitHub 生成 Token：Settings → Developer settings → Personal access tokens (classic)
  - **Note**：填写用途说明
  - **Expiration**：选择 No expiration（永不过期）
  - **Scopes**：勾选 repo（私有仓库全控制）
  - 点击 Generate token，复制 token 到手机端粘贴配置

#### 8.10 手机端同步效果与验证
- 手机创建笔记，等待自动推送，GitHub 网页端验证同步成功
- 电脑端自动 pull，确保双向一致
- 手机与电脑双向自动同步，笔记保持一致

#### 8.11 使用注意事项
- 手机与电脑最好不要同时编辑同一个文件，避免 Git 冲突
- 冲突时需手动解决

#### 8.12 多重备份策略
- **GitHub**：云备份（热备份）
- **网盘**：定时备份笔记文件夹
- **硬盘**：冷备份
- 组合使用，保证笔记永远不会丢失

### 第九章：图片管理插件 —— Custom Attachment Location（第三方）

#### 9.1 Obsidian 默认图片存储的问题
- 粘贴图片后文件散落在笔记同级目录，根目录杂乱
- 使用非标准的 Wiki 链接语法 `[[图片名]]`，GitHub 网页端和 VS Code 无法显示

#### 9.2 插件安装
- 在社区插件市场搜索安装 **Custom Attachment Location**（RainCat1998）

#### 9.3 插件核心配置
- **Markdown URL 格式**：`assets/${noteFileName}/${generatedAttachmentFileName}`（生成标准 Markdown 链接 `![ ]()`，兼容 GitHub 和 VS Code）
- **附件存放位置**：`./assets/${noteFileName}`（按笔记名分子文件夹）
- **生成文件名**：`file-${date:YYYYMMDDHhmmss}`（时间戳命名）
- **附件重命名**：开启后，笔记重命名/移动时自动同步附件

#### 9.4 插件效果
- 图片有序存储到 `assets/笔记名` 子文件夹，根目录保持整洁
- 链接为标准 Markdown 格式，跨工具正常渲染
- 重命名或移动笔记时，附件文件夹和链接自动同步更新

#### 9.5 图片调整技巧
- 图片太宽时，在方括号内输入数字调整大小（如 `![500](图片路径)`）

### 第十章：笔记导出插件 —— Enhancing Export（第三方）

#### 10.1 插件介绍
- 安装 **Enhancing Export** 插件，基于 Pandoc 增强导出能力
- 支持导出为 HTML、DOCX、ePub、PDF、Hugo 等格式

#### 10.2 Pandoc 安装与配置
- 从 GitHub jgm/pandoc Releases 下载对应系统版本
- 解压到指定目录，在插件配置中粘贴 Pandoc 可执行文件路径

#### 10.3 导出操作
- 右键笔记 → 导出为 → 选择目标格式（如 Word .docx）
- 导出后文档内配图正常显示

### 第十一章：其他常用第三方插件介绍与用法

#### 11.1 插件管理基础
- 核心插件 vs 社区插件
- 开启社区插件市场的方法
- 插件的安装、启用、更新与卸载

#### 11.2 数据查询插件 —— Dataview
- **功能**：像查询数据库一样搜索和展示笔记
- **场景**：汇总带特定标签的笔记、按属性筛选、生成动态目录
- **基础用法**：用 Dataview 查询语言（DQL）或 JavaScript 查询笔记元数据

#### 11.3 模板增强插件 —— Templater
- **功能**：比内置模板更强大的动态模板系统
- **场景**：自动插入当前日期、时间、文件名、动态生成内容
- **配合**：常与 QuickAdd 插件联用，一键创建标准化笔记

#### 11.4 日历与日记插件 —— Calendar
- **功能**：可视化日历面板，快速创建和查看每日笔记
- **场景**：习惯追踪、每日复盘、日程记录

#### 11.5 快速添加插件 —— QuickAdd
- **功能**：通过快捷键或按钮快速执行预设操作
- **场景**：一键创建特定类型的笔记、快速添加待办、捕获灵感

#### 11.6 其他实用插件推荐
- **Recent Files**：显示最近打开的笔记
- **Outliner**：大纲式编辑增强，支持折叠、拖拽
- **Excalidraw**：手绘风格画图，集成到 Obsidian 中
- **PDF Plus**：在 PDF 上标注并生成可跳转的笔记链接

### 第十二章：AI 接入与智能化工作流（第三方工具）

#### 12.1 为什么用 AI 编程工具而非社区 AI 插件
- 一线大厂出品，功能强、模型适配度好
- 与文件系统交互是核心能力，处理 Markdown 笔记得心应手

#### 12.2 Gemini CLI 安装与配置
- 安装 Node.js 环境
- 执行 `npm install -g @google/gemini-cli` 安装
- 启动后登录 Google 账号完成授权

#### 12.3 进阶玩法案例
- **智能选题**：分析历史笔记，调用网络搜索生成爆款选题并输出 Markdown 文件
- **批量文件处理**：自动创建文件夹结构，批量生成大纲文件
- **模仿文风写作**：学习历史笔记风格，生成视频脚本等长文本

#### 12.4 Git 保障 AI 操作安全
- 每一次 AI 修改都被 Git 记录，不满意可随时回滚
- 可放心让 AI 批量操作文件，不怕搞丢或搞坏
